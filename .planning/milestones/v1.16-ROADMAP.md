## Overview

Directus-backed digital signage CMS inside the existing KPI Dashboard monorepo. Adds an admin UI for managing Media/Playlists/Devices, a bundle-isolated Chromium-kiosk player for Raspberry Pi, tag-based playlist-to-device routing, SSE live updates, PPTX conversion, one-script Pi provisioning, and an offline-resilient Python sidecar on each kiosk. Small-fleet scope (≤5 devices), single-site, cache-and-loop offline.

47 requirements across DB/BE/SCH/ADM/PLY/DIFF/OPS/INF — all landed.

## Key accomplishments

- **Admin-UI at `/signage`** — bilingual (DE "du" tone) Media / Playlists / Devices tabs, drag-reorder editor, WYSIWYG preview, tag picker.
- **Kiosk player bundle** — 210 KB gz, PWA-precached shell, 6-digit pairing flow, EventSource + 45 s watchdog + 30 s polling fallback, 6 format handlers (image/video/pdf-crossfade/url/html/pptx).
- **Backend schema + API** — 8-table Alembic migration, scoped device JWT (HS256 24 h TTL), SSE fan-out via in-process `asyncio.Queue`, tag-to-playlist resolver with ETag, APScheduler singleton for cron + PPTX + heartbeat-sweeper.
- **PPTX pipeline** — async subprocess + 60 s timeout + semaphore + state machine + startup stuck-reset + `libreoffice-impress` apt layer with Carlito/Caladea/Noto/DejaVu fonts.
- **Pi provisioning** — single `scripts/provision-pi.sh` takes Bookworm Lite 64-bit to paired, playing kiosk; dedicated non-root `signage` user; labwc + Chromium wayland kiosk via systemd user services.
- **Offline-resilient sidecar** — `pi-sidecar/` FastAPI app proxy-caches `/api/signage/player/playlist` + media bytes to `/var/lib/signage/` (0700); 5-minute Wi-Fi drop keeps the loop running; auto-reconnect ≤30 s.
- **Bilingual admin guide + operator runbook** — new article at `frontend/src/docs/{en,de}/admin-guide/digital-signage.md`; runbook at `docs/operator-runbook.md` with verbatim systemd units + SGN-OPS-03 Chromium flag set + 8 recovery procedures.
