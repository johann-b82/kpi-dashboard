## Component Inventory

### New Component: SegmentedControl

**File:** `frontend/src/components/ui/segmented-control.tsx`

**Props (TypeScript):**

```ts
interface SegmentedControlProps<T extends string> {
  segments: Array<{ value: T; label: string }>;
  value: T;
  onChange: (value: T) => void;
  disabled?: boolean;
  'aria-label'?: string;
}
```

**Structure:**

```
<div role="radiogroup" class="container">
  {segments.map(seg => (
    <button role="radio" aria-checked={...} class="segment" />
  ))}
</div>
```

**Container classes:** `inline-flex items-center bg-muted rounded-full p-1 gap-0`

**Active segment classes:** `bg-foreground text-background text-sm font-medium rounded-full px-3 h-6 transition-colors`

**Inactive segment classes:** `transparent text-muted-foreground text-sm font-normal rounded-full px-3 h-6 hover:text-foreground transition-colors`

**Disabled state:** Apply `opacity-50 pointer-events-none` to the container when `disabled={true}`. The wrapping element receives `aria-disabled="true"`.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill internal gap (`p-1`) |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**SegmentedControl-specific:**
- Container padding: `p-1` (4px) — creates gap between container edge and pill
- Segment horizontal padding: `px-3` (12px) per side
- Segment height: `h-6` (24px) — equivalent to `h-8` (32px) total container height including 4px container padding top and bottom
- Gap between segments: `gap-0` — segments are flush; container padding provides visual separation

Exceptions: NavBar LanguageToggle container height constrained by NavBar `h-16` (64px); no changes to NavBar height.

| Role | Token | Light Mode Value | Usage |
|------|-------|-----------------|-------|

## Interaction Contract

### State Machine per SegmentedControl

```
IDLE
  └─ click inactive segment → onChange(value) fired → parent updates value prop → active segment re-renders
  └─ hover inactive segment → text transitions to text-foreground (150ms ease)
  └─ disabled=true → opacity-50, pointer-events-none, aria-disabled="true"
```

### Navigation Semantics (Sales/HR control)

- Uses `<button>` elements with `role="radio"` + `aria-checked`
- `onClick` calls wouter `navigate()` — URL is source of truth
- Active state derived from `useLocation()` — not local state
- No right-click "open in new tab" (accepted trade-off per CONTEXT.md D-05)

### Language Toggle Disabled State

- NavBar LanguageToggle SegmentedControl receives `disabled={isDirty || mutation.isPending}`
- When disabled: `opacity-50 pointer-events-none`, container gets `title={t("settings.preferences.toggle_disabled_tooltip")}`
- Preserved exactly per Phase 7 D-13

### Transition

- Segment text color: `transition-colors duration-150 ease-in-out`
- No animated sliding pill — instant state switch on active segment change (per REQUIREMENTS.md "Out of Scope")

### Keyboard

- Arrow key navigation: out of scope for v1.5 (per REQUIREMENTS.md)
- `Tab` focuses the control group, `Space`/`Enter` activates focused segment — native button behavior

## Implementation Notes for Executor

1. **Generic typing:** Make `SegmentedControlProps` generic over `T extends string` so TypeScript enforces that `value` and `segments[].value` share the same union type. This prevents mismatched string comparisons at each consumer.

2. **No new context or hooks needed:** Each consumer already has access to its active-state source. Wire directly.

3. **i18n labels:** Pass pre-translated strings as `label` in the `segments` array (i.e., call `t(key)` at the consumer, not inside SegmentedControl). This keeps the component library-agnostic.

4. **PreferencesCard migration:** The existing `role="radiogroup"` pattern in `PreferencesCard.tsx` is already close to the target. Replace the inline `<div role="radiogroup">` + `<button>` structure with `<SegmentedControl>`.

5. **NavBar layout:** Sales/HR SegmentedControl replaces the two `<Link>` elements. Language SegmentedControl replaces `<LanguageToggle>`. NavBar layout structure (`max-w-7xl mx-auto px-6 h-full flex items-center gap-6` + `ml-auto`) is unchanged.
