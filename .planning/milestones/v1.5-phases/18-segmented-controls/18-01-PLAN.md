Create `frontend/src/components/ui/segmented-control.tsx` with:

**Interface (per UI-SPEC):**
```ts
interface SegmentedControlProps<T extends string> {
  segments: Array<{ value: T; label: string }>;
  value: T;
  onChange: (value: T) => void;
  disabled?: boolean;
  'aria-label'?: string;
}
```

**Component structure:**
- Outer `<div>` with `role="radiogroup"` and optional `aria-label` prop pass-through
- When `disabled={true}`: add `opacity-50 pointer-events-none` to container and `aria-disabled="true"`
- Map over `segments` array rendering `<button>` elements with `role="radio"` and `aria-checked={segment.value === value}`
- Each button `type="button"`, `onClick` calls `onChange(segment.value)`

**Exact Tailwind classes (per UI-SPEC, sourced from D-01, D-02, D-03, D-06):**

Container: `inline-flex items-center bg-muted rounded-full p-1 gap-0`

Active segment: `bg-foreground text-background text-sm font-medium rounded-full px-3 h-6 transition-colors`

Inactive segment: `transparent text-muted-foreground text-sm font-normal rounded-full px-3 h-6 hover:text-foreground transition-colors`

Note: `transition-colors` provides `duration-150ms ease` timing per UI-SPEC interaction contract. No animated sliding pill — instant active state switch per REQUIREMENTS.md "Out of Scope".

**Hover behavior:** Inactive segments transition text color `text-muted-foreground` to `text-foreground` on hover. No background color change on hover.

**Export:** Named export `SegmentedControl` (not default export — matches project convention).

**Disabled tooltip:** When `disabled={true}`, the container should accept a `title` prop. Add `title?: string` to the props interface to support the NavBar language toggle tooltip pattern (D-09).

Do NOT add keyboard arrow-key navigation — out of scope per REQUIREMENTS.md. Native `Tab` + `Space`/`Enter` button behavior is sufficient.
