## Overview

Closed the v1.17 operator carry-forwards (hardware E2E Scenarios 4 + 5 on a `provision-pi.sh`-provisioned Pi; player bundle back under 200 KB gz via dynamic-import of `PdfPlayer` + `react-pdf`), then added time-based playlist scheduling: a new `signage_schedules` table (weekday_mask + start_hhmm + end_hhmm + priority + enabled), a time-window-aware resolver that falls back to the existing tag-to-playlist resolution, an admin UI 4th tab with bilingual docs, and analytics-lite uptime/missed-heartbeat badges on the Devices table. Mid-milestone scope change retired the `.img.xz` custom-image pipeline entirely — SGN-POL-01/02/03/06 dropped, Pi provisioning is now single-path via `scripts/provision-pi.sh` on fresh Bookworm Lite 64-bit.

## Key accomplishments

- **Player bundle back under 200 KB gz** — `PdfPlayer` + `react-pdf` lazy-loaded via LAZY_PREFIXES allowlist; entry chunk budget reset from 210 000 to 200 000.
- **Scenarios 4 + 5 hardware E2E PASS** on v1.18 Pi — reconnect → admin-mutation ≤ 30 s; sidecar restart visual continuity + `/health` ≤ 15 s. Documented in `50-E2E-RESULTS.md`.
- **Time-aware playlist resolver** — `signage_schedules` migration (7-bit weekday_mask, `start_hhmm < end_hhmm`, priority, enabled), `app_settings.timezone` (default `Europe/Berlin`), extracted `_build_envelope_for_playlist` helper so scheduled and tag-based envelopes are byte-identical (ETag invariant preserved). 7 integration tests cover the full decision matrix.
- **Schedule admin UI** — 4th `/signage/schedules` tab under `<AdminOnly>`, `ScheduleEditDialog` with D-07/11/12 validation tree, weekday checkbox row, two HH:MM inputs + priority + enabled, `ScheduleDeleteDialog` with RESTRICT caveat, cross-tab 409 deep-link from PlaylistsPage with scrollIntoView + ring highlight. SSE `schedule-changed` fanout + new `useAdminSignageEvents` hook.
- **Analytics-lite** — append-only `signage_heartbeat_event` log (composite PK `(device_id, ts)`, 25 h retention via existing sweeper), read-only analytics endpoint, UptimeBadge with green/yellow/red thresholds, two new Devices table columns with visibility-change + 30 s polling.
- **Custom-image pipeline retirement** — `pi-image/` directory, `pi-image.yml` workflow, minisign scaffold, and `signage-packages.txt`↔`00-packages-nr` drift-check removed; SGN-POL-01/02/03/06 dropped. Single Pi-provisioning path simplifies operator flow.
- **Bilingual admin-guide parity held** — Schedules/Zeitpläne and Analytics/Analyse sections added, 55 EN == 55 DE signage keys, DE du-tone verified (0 Sie/Ihre/Ihr).
