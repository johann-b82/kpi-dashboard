## Outcome notes

- **11 / 11 active requirements landed.** All SGN-POL-04/05, SGN-TIME-01..04, SGN-SCHED-UI-01..04, and SGN-ANA-01 have code + tests + doc evidence in the v1.18 phase directories.
- **Scope change 2026-04-21 — custom image pipeline retired.** SGN-POL-01 (minisign key ceremony), SGN-POL-02 (arm64 self-hosted runner), SGN-POL-03 (tag-triggered signed-image release), and SGN-POL-06 (image↔provision byte-diff test) were **dropped** along with the `pi-image/` directory, `.github/workflows/pi-image.yml`, and the `signage-packages.txt`↔`00-packages-nr` drift-check. The Pi player ships exclusively via `scripts/provision-pi.sh` on fresh Raspberry Pi OS Bookworm Lite 64-bit.
- **SGN-POL-04 closure note:** Scenarios 4 + 5 PASS on v1.18 Pi (Bookworm Lite 64-bit, provisioned via `scripts/provision-pi.sh`). Thresholds (reconnect → admin-mutation ≤ 30 s; sidecar restart visual continuity + `/health` ≤ 15 s) verified by direct observation; exact numerical timings documented as "not recorded" in `50-E2E-RESULTS.md`.
- **SGN-ANA-01 amendment:** added `signage_heartbeat_event` append-only log (composite PK `(device_id, ts)`, 25 h retention pruned by the existing 60 s heartbeat sweeper) so the 60-second-window uptime metric is measurable. Amendment recorded 2026-04-21 per Phase 53 CONTEXT D-01.

**Core Value:** Close v1.17's operator carry-forwards and add time-based playlist scheduling so operators can run "Menu A 07:00–11:00, Menu B 11:00–14:00, Weekend menu 10:00–16:00 Sat/Sun" without touching the device.

**Locked defaults (2026-04-21):** weekday_mask + HH:MM window (not iCal RRULE); heartbeat + last-seen analytics only (no per-item playtime); per-device calibration deferred to v1.19; Pi provisioning single-path via `scripts/provision-pi.sh`.

### Schedule schema + resolver (SGN-TIME-*)

- [x] **SGN-TIME-01** — *validated*. Alembic migration creates `signage_schedules` with all documented columns and CHECK constraints (`weekday_mask BETWEEN 0 AND 127`, `0 <= start_hhmm <= 2359`, `0 <= end_hhmm <= 2359`, `start_hhmm < end_hhmm`). `app_settings.timezone` added with `server_default='Europe/Berlin'`. Round-trip upgrade/downgrade clean.
- [x] **SGN-TIME-02** — *validated*. Resolver gains time-window awareness; picks highest `(priority DESC, updated_at DESC)` schedule matching weekday + time-of-day + tag overlap. Falls back to always-on tag resolution when no schedule matches. Pure-read (D-10). Timezone from `app_settings.timezone`. `_build_envelope_for_playlist` helper ensures byte-identical envelopes (ETag invariant preserved). Weekday bit-test uses SQLAlchemy `bindparam` (no f-string SQL).
- [x] **SGN-TIME-03** — *validated*. 7 integration tests cover schedule-match, priority tiebreak, weekday-miss, time-miss, disabled-schedule-skip, tag-mismatch-skip, empty-schedules-falls-back-to-tag-resolver.
- [x] **SGN-TIME-04** — *validated*. Schedule mutations fire `notify_device` SSE fanout per `(device, playlist_id)` pair; PATCH union case sends one event per playlist. Playlist DELETE returns 409 with `schedule_ids` when referenced by active schedules (mirrors existing media 409 shape).

### Schedule admin UI (SGN-SCHED-UI-*)

### Analytics-lite (SGN-ANA-*)

- [x] **SGN-ANA-01** — *validated (adjusted)*. Admin Devices table gains Uptime-24h % and Missed-windows-24h badges (green ≥ 95 %, yellow 80–95 %, red < 80 %).
