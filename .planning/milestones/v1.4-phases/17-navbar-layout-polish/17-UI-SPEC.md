## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon inner gaps, inline padding within badges |
| sm | 8px | Gap between preset buttons in DateRangeFilter (gap-2) |
| md | 16px | Default horizontal element spacing (gap-4 in navbar action area) |
| lg | 24px | Horizontal page padding (px-6 on navbar and sub-header containers) |
| xl | 32px | — |
| 2xl | 48px | Sub-header height (h-12 = 48px) |
| 3xl | 64px | Navbar height (h-16 = 64px, locked per D-02) |

Exceptions:
- Logo: `max-h-8 max-w-8` (32px, locked per D-01) — visual brand anchor, not a content spacing unit
- Logo-to-text gap: `gap-6` (24px) — matches existing navbar item gap, source: NavBar.tsx line 34
- Content top offset: `pt-28` (112px = navbar 64px + sub-header 48px) — must exactly equal combined fixed header height
- Icon-only touch targets: `p-2` (8px padding on 20px icon = 36px effective; acceptable for desktop-first, 1080p+ target per REQUIREMENTS.md)

| Body | 14px (text-sm) | 400 (regular) | 1.5 | Default UI text, tab labels, button labels |

## Color

All values are CSS custom properties from `frontend/src/index.css`. Do not use hardcoded hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` | Page surface below fixed headers |
| Secondary (30%) | `bg-card` | Navbar (`bg-card` — existing), sub-header (`bg-card` — matches navbar material) |
| Accent (10%) | `text-primary` / `border-primary` | Specific reserved elements only (see list below) |
| Destructive | `text-destructive` / `bg-destructive` | Not applicable to this phase — no destructive actions |
| Muted | `text-muted-foreground` | FreshnessIndicator timestamp text |
| Border | `border-border` | Navbar bottom border, sub-header bottom border, separator line |
| Hover state | `hover:bg-accent/10` | Icon-only link hover background (upload icon, settings icon) |

Accent (`text-primary` / `border-primary`) reserved for:
1. Active tab text: `text-primary font-semibold` on the currently active Sales or HR tab
2. Active tab underline: `border-b-2 border-primary` on the currently active tab link
3. Upload icon active state: `text-primary` when `location === "/upload"` (matches D-07)
4. Settings icon active state: `text-primary` when `location === "/settings"` (existing pattern)
5. Active preset button: `variant="default"` (shadcn Button — renders with primary background) when selected

Accent is NOT used for: inactive tabs, hover states on tabs (hover uses `hover:text-primary` only as transition target), the sub-header background, or the freshness timestamp.

## Interaction Contract

### Tab Active State

- Trigger: `location === "/"` (Sales) or `location === "/hr"` (HR)
- Visual: `border-b-2 border-primary font-semibold text-primary pb-1`
- No animation — direct class swap via `useLocation()`
- Source: D-14, D-15 (already implemented; no change needed)

### Upload Icon — Navbar Action Area

- Element: `<Link href="/upload" aria-label={t("nav.upload")} className={uploadLinkClass}>`
- Icon: `<UploadIcon className="h-5 w-5" />` (lucide-react `Upload`)
- Active (on `/upload`): `text-primary`
- Inactive: `text-foreground`
- Hover: `hover:bg-accent/10` (8px padding, rounded-md)
- No button wrapper — styled Link directly (constraint: no `<Button>` inside `<Link>`, see NavBar.tsx comment)
- Source: D-05, D-06, D-07

### Settings Icon — Navbar Action Area (unchanged)

- Existing `settingsLinkClass` pattern — no change
- Documented here for reference only

### DateRangeFilter Preset Buttons — SubHeader

- Render: only when `location === "/"`
- State owner: `DateRangeContext` (via `useDateRange()`)
- Active preset button: `variant="default"` (shadcn Button)
- Inactive preset buttons: `variant="outline"` (shadcn Button)
- Size: `size="sm"` — preserves existing sizing
- Gap between buttons: `gap-2` (8px) — preserves existing spacing
- State drives: `KpiCardGrid` and `RevenueChart` in `DashboardPage` via context consumption

### FreshnessIndicator — SubHeader

- Render: always (all routes — `/`, `/hr`, `/upload`, `/settings`)
- Alignment: right-aligned via `justify-between` on SubHeader's inner flex container
- Styling: `text-xs text-muted-foreground` (unchanged from existing component)
- No route gating — `showUploadFreshness` guard removed from NavBar

## Copywriting Contract

| Element | EN | DE |
|---------|----|----|
| Upload icon tooltip / aria-label | "Upload" | "Upload" |
| Preset: this month | Existing `dashboard.filter.thisMonth` key | Existing key |
| Preset: this quarter | Existing `dashboard.filter.thisQuarter` key | Existing key |
| Preset: this year | Existing `dashboard.filter.thisYear` key | Existing key |
| Preset: all time | Existing `dashboard.filter.allTime` key | Existing key |
| Freshness: has data | Existing `nav.lastUpdated` key + formatted date | Existing key |
| Freshness: no data | Existing `nav.lastUpdated.never` key | Existing key |

No new i18n keys are required for this phase. `nav.upload` already exists in both `en.json` and `de.json` with value "Upload".

If any new strings are added during implementation (e.g., a sub-header `aria-label`), they must be added to BOTH locale files simultaneously before committing.

Empty state: Not applicable — FreshnessIndicator renders its own "never updated" state using the existing `nav.lastUpdated.never` key.

Error state: Not applicable — this phase introduces no new data-fetching. FreshnessIndicator handles its own loading/error states internally.

Destructive actions: None in this phase.

Primary CTA: Not applicable — this is a navigation/layout phase with no primary action surface.
