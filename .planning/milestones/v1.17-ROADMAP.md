## Overview

Pre-bake the Phase 48 signage stack into a flashable Raspberry Pi OS Bookworm Lite 64-bit image. Operator flashes one `.img.xz` via Raspberry Pi Imager, boots the Pi, and lands on the 6-digit pairing code — no SSH, git, or apt steps between flash and pairing.

**Base:** RPi OS Bookworm Lite 64-bit (not Trixie — research `.planning/research/debian-trixie-pi-image.md` returned CONDITIONAL-GO, deferred). **GUI:** thin (labwc + Chromium kiosk only, no desktop). **Builder:** pi-gen as git submodule. **Signing:** minisign (Ed25519). **Distribution:** GitHub Releases via self-hosted arm64 runner.

## Key accomplishments

- **Runtime path proven on real Pi 4 hardware** — `scripts/provision-pi.sh` takes a vanilla Bookworm Lite image to a paired, playing kiosk. Scenarios 1–3 of the E2E walkthrough PASS. Supersedes Phase 48's deferred hardware walkthrough.
- **Baked-image CI path CODE-COMPLETE** — pi-gen fork at `pi-image/`, signing docs, release workflow, release template all committed.
- **Installer library refactor** — `scripts/lib/signage-install.sh` (7 shared functions) + `scripts/lib/signage-packages.txt` (SSOT) with CI drift-check; `provision-pi.sh` and the pi-gen stage source the same code.
- **3 systemd-unit defects found and fixed on hardware** — `signage-player.service` WantedBy, Wayland socket name, `labwc.service` circular dependency. Fixes propagate to both runtime and image-build paths via shared templates.
