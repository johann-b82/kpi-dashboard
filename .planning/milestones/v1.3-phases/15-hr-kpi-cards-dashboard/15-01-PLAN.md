**2. Create `backend/app/services/hr_kpi_aggregation.py`** with the following functions:

**Helper: `_month_bounds(year, month) -> tuple[date, date]`**
- Uses `calendar.monthrange(year, month)[1]` to get last day
- Returns `(date(year, month, 1), date(year, month, last_day))`

**Helper: `_prev_month(year, month) -> tuple[int, int]`**
- Returns `(year - 1, 12)` when `month == 1`, else `(year, month - 1)`
- Per Pitfall 4 in RESEARCH.md — prevents January underflow

**Helper: `_headcount_at_eom(session, last_day, department=None) -> int`**
- Per D-02: count employees where `hire_date <= last_day AND (termination_date IS NULL OR termination_date > last_day)`
- When `department` is not None, add `WHERE department = department` (exact string match per D-11)
- Uses `select(func.count(PersonioEmployee.id))` with appropriate `.where()` clauses

**Function: `_overtime_ratio(session, first_day, last_day) -> float | None`** (HRKPI-01)
- Join `PersonioAttendance` with `PersonioEmployee` on `employee_id`
- Filter attendance records where `date >= first_day AND date <= last_day`
- For each row: `worked = (end_time.hour*60 + end_time.minute - start_time.hour*60 - start_time.minute - break_minutes) / 60.0`
- Daily quota: `float(weekly_working_hours) / 5.0` (if weekly_working_hours is not None, else skip overtime calc for that record)
- Overtime for that record: `max(0.0, worked - daily_quota)`
- Sum all overtime_hours / sum all total_hours. Return None if total_hours == 0
- Per D-01: uses calendar month window

**Function: `_sick_leave_ratio(session, first_day, last_day, sick_leave_type_id) -> float | None`** (HRKPI-02)
- Query `PersonioAbsence` where `absence_type_id = sick_leave_type_id` AND `start_date <= last_day AND end_date >= first_day` (overlap logic per Pitfall 1)
- For each absence: clip to `[max(start_date, first_day), min(end_date, last_day)]` to get clipped days
- Compute sick hours per absence:
  - If `time_unit == 'hours'` and `hours is not None`: use `hours` value, prorated by `clipped_days / total_absence_days`
  - Else: join with employee's `weekly_working_hours / 5.0` per day (default 8.0 if null), multiply by clipped_days
- Denominator: sum of all employees' `weekly_working_hours / 5.0 * working_days_in_month` where working_days_in_month is approximated as the count of weekdays (Mon-Fri) in `[first_day, last_day]`
  - Query all active employees at end of month, sum their daily hours * weekday count
  - Simpler: `headcount * avg_daily_hours * weekday_count` — but per-employee is more accurate since weekly_working_hours varies
  - Use: iterate active employees, sum `(weekly_working_hours or 40.0) / 5.0 * weekday_count`
- Return sick_hours / total_scheduled_hours, or None if denominator is 0

**Function: `_fluctuation(session, first_day, last_day) -> float | None`** (HRKPI-03)
- Count employees with `termination_date >= first_day AND termination_date <= last_day`
- Headcount = `_headcount_at_eom(session, last_day)`
- Return leavers / headcount, or None if headcount == 0
- Per D-02: denominator is end-of-month snapshot

**Function: `_skill_development(session, last_day, skill_attr_key) -> float | None`** (HRKPI-04)
- Per RESEARCH.md recommendation (a): employees with non-null value for `raw_json['attributes'][skill_attr_key]['value']`
- Use SQLAlchemy JSONB: `PersonioEmployee.raw_json["attributes"][skill_attr_key]["value"]` with `.as_string() != "null"` or `.isnot(None)` check
- Filter active employees only (hire_date <= last_day, termination_date null or > last_day)
- Denominator: total headcount at end of month
- Return skilled_count / headcount, or None if headcount == 0
- Add code comment: "Proxy metric: employees with current non-null value for the configured attribute. No historical snapshot table exists."

**Function: `_revenue_per_production_employee(session, first_day, last_day, production_dept) -> float | None`** (HRKPI-05)
- Revenue numerator: `SUM(SalesRecord.total_value) WHERE total_value > 0 AND order_date >= first_day AND order_date <= last_day` — reuse `aggregate_kpi_summary(session, first_day, last_day)`, extract `total_revenue` if result is not None. Per D-12.
- Denominator: `_headcount_at_eom(session, last_day, department=production_dept)` — production employees only. Per D-13.
- Return `float(revenue) / headcount` if both exist, else None. Per D-14 and Pitfall 5.

**Main entry point: `async def compute_hr_kpis(db: AsyncSession) -> HrKpiResponse`**
- Load AppSettings singleton: `db.execute(select(AppSettings).where(AppSettings.id == 1))` — `.scalar_one_or_none()`
- Compute current month bounds from `date.today()`: `_month_bounds(today.year, today.month)`
- Compute previous month bounds: `_month_bounds(*_prev_month(today.year, today.month))`
- Compute previous year same month bounds: `_month_bounds(today.year - 1, today.month)`
- For each of the three windows, compute all 5 KPIs using **sequential awaits** (per Pitfall 2 — NO asyncio.gather on same session)
- Build HrKpiResponse:
  - `overtime_ratio`: always configured, compute for all 3 windows
  - `sick_leave_ratio`: `is_configured = settings.personio_sick_leave_type_id is not None`. If not configured, return `HrKpiValue(value=None, is_configured=False)`. Per D-08.
  - `fluctuation`: always configured, compute for all 3 windows
  - `skill_development`: `is_configured = settings.personio_skill_attr_key is not None`. If not configured, return `HrKpiValue(value=None, is_configured=False)`. Per D-08.
  - `revenue_per_production_employee`: `is_configured = settings.personio_production_dept is not None`. If not configured, return `HrKpiValue(value=None, is_configured=False)`. Per D-08.
- If settings is None (no AppSettings row), treat all configurable KPIs as not configured
- Return HrKpiResponse with all 5 KPI values populated

**1. Create `backend/app/routers/hr_kpis.py`:**

```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_async_db_session
from app.schemas import HrKpiResponse
from app.services.hr_kpi_aggregation import compute_hr_kpis

router = APIRouter(prefix="/api/hr", tags=["hr-kpis"])

@router.get("/kpis", response_model=HrKpiResponse)
async def get_hr_kpis(
    db: AsyncSession = Depends(get_async_db_session),
) -> HrKpiResponse:
    """Return all 5 HR KPIs for the current calendar month with prior-period comparisons.

    Each KPI includes value, is_configured, previous_period, and previous_year.
    KPIs requiring unconfigured settings return is_configured=False, value=None.
    When no Personio data exists, all values are None with is_configured=True.
    Per D-03: no date parameters — server computes fixed calendar month windows.
    """
    return await compute_hr_kpis(db)
```

This mirrors the pattern in `backend/app/routers/kpis.py` — single GET endpoint, single service call, response_model for serialization. Per D-03 (no date filter on HR tab), there are no query parameters.
