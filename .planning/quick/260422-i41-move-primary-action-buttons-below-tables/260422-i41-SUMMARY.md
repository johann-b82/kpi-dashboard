# Quick Task 260422-i41: Move Primary Action Buttons Below Tables on Signage Pages — Summary

**One-liner:** Relocated the page-level primary CTA from above to below the list on all four signage admin pages (media, playlists, devices, schedules), establishing a consistent "list first, action below" pattern right-aligned across the surface.

## What Shipped

All four Digital Signage list pages now render their primary CTA below the list (right-aligned), matching user's requested visual pattern:

| Page | CTA Label | Gating | Handler (preserved) |
|------|-----------|--------|----------------------|
| /signage/media | `signage.admin.media.register_url_button` ("URL / HTML registrieren") | Always rendered | `setRegisterOpen(true)` → MediaRegisterUrlDialog |
| /signage/playlists | `signage.admin.playlists.new_button` ("Neue Playlist") | `playlists.length > 0` | `setNewOpen(true)` → PlaylistNewDialog |
| /signage/devices | `signage.admin.devices.pair_button` ("Gerät koppeln") | List-path only (empty state early-returns) | `setLocation("/signage/pair")` |
| /signage/schedules | `signage.admin.schedules.new_cta` ("Neuer Zeitplan") | `!isEmpty` | `setEditing(null)` → ScheduleEditDialog (create) |

No handler, label, variant, size, disabled-logic, or i18n-key changes. Pure structural relocation within each JSX tree.

## Relocation Details

### MediaPage.tsx
- **Removed from:** above the `{mediaQuery.data && mediaQuery.data.length > 0 && ( … grid … )}` block.
- **Inserted at:** after the grid block, before `<MediaRegisterUrlDialog … />`.
- **Gate:** none (always rendered). The drop-zone's internal "Dateien auswählen" button remains in place.

### PlaylistsPage.tsx
- **Removed from:** above the loading/error/empty/table ternary (~lines 131–135 pre-move).
- **Inserted at:** after the ternary, before `<PlaylistNewDialog … />`.
- **Gate:** `{playlists.length > 0 && ( … )}` — empty-state card keeps its own CTA; avoids duplicate.

### DevicesPage.tsx
- **Removed from:** above the devices-table wrapper (~lines 125–129 pre-move), wrapper class was `flex items-center justify-between`.
- **Inserted at:** after the `<div className="rounded-md border border-border bg-card"> … </div>` table wrapper, before `<DeviceEditDialog … />`.
- **Wrapper class normalized:** `flex justify-end` (legacy `justify-between` existed for a historical two-item layout — only ever held one button).
- **Gate:** none in the list-path; empty-state early-return (lines ~99–116) already owns its own CTA.

### SchedulesPage.tsx
- **Removed from:** above the `{isEmpty ? … : <table> }` ternary (~lines 216–222 pre-move).
- **Inserted at:** after the ternary, before `<ScheduleEditDialog … />`.
- **Gate:** `!isEmpty` preserved.
