# Quick Task 260417-e45: Rename Sales Dashboard Route to /sales Summary

Renamed the Sales Dashboard route from `/` to `/sales` and made bare `/` redirect to `/home` (App Launcher) using wouter's `<Redirect>` component; propagated the new path across NavBar (SegmentedControl, back-button, sessionStorage guard, Dashboard type), SubHeader (DateRangeFilter gate), LauncherPage (KPI tile onClick), and refreshed stale JSDoc comments in AuthGate and LoginPage.

## What Changed

### Task 1 — App.tsx route table (commit `f41ddbe`)

- Added `Redirect` to the wouter import.
- `<Route path="/" component={DashboardPage} />` → `<Route path="/sales" component={DashboardPage} />`.
- Inserted `<Route path="/"><Redirect to="/home" /></Route>` immediately after the `/sales` route. On mount, wouter's `<Redirect>` calls `setLocation("/home")`, so any bare `/` visit bounces straight to the App Launcher.

### Task 2 — Consumer propagation (commit `d6f5151`)

- **NavBar.tsx** — 6 edits: `Dashboard` type is now `"/sales" | "/hr"`; `getLastDashboard()` default returns `"/sales"`; the `useEffect` guard compares against `"/sales"`; the fallback-comment mentions `"/sales"`; the SegmentedControl Sales segment `value` is `"/sales"`; and the `value=` ternary fallback is `"/sales"`.
- **SubHeader.tsx** — 1 edit: DateRangeFilter render gate is `location === "/sales"` (no change to the `/home` null-guard or `/hr` branch).
- **LauncherPage.tsx** — 2 edits: KPI Dashboard tile `onClick` calls `setLocation("/sales")` and the comment above it reflects the new path.
- **AuthGate.tsx** — JSDoc-only: `Authed on /login → redirect to /home.` (runtime `setLocation("/home")` was already correct — only the comment was stale).
- **LoginPage.tsx** — JSDoc-only: `On success, AuthGate handles navigation to /home.` (no routing in this file).

No i18n strings (`nav.sales` label stays), no CSS/className, and no new imports were changed.
