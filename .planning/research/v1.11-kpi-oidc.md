# OIDC Authentication Research — KPI Light v1.11

**Date:** 2026-04-14
**Scope:** Adding OIDC login against a Dex IdP to FastAPI 0.135 + React 19 + wouter stack
**Confidence:** MEDIUM–HIGH (core patterns well-established; Dex-specific logout needs runtime verification)

---

## 1. Python OIDC Library Verdict

### Recommendation: `authlib` (1.6.x) via Starlette integration

**Why authlib wins for this project:**

| Library | Verdict | Reason |
|---|---|---|
| `authlib` | **USE THIS** | Full OIDC client built-in, async-native via Starlette integration, handles PKCE + discovery + token validation, active maintenance (1.6.10 released April 2026), first-class FastAPI docs |
| `fastapi-oidc` (PyPI) | Avoid | Narrowly scoped — only validates externally-issued tokens, does not handle login/callback flows. Useful as a resource-server lib only |
| `fastapi-users` | Avoid | Brings its own user management, router, database layer (heavy). No external IdP OIDC support; only OAuth social login via httpx. Wrong tool for delegating to Dex |
| `httpx` + `python-jose` | Avoid | DIY: implement PKCE, state/nonce, token validation, JWKS caching manually. 400+ lines for what authlib gives in 30. `python-jose` is also maintenance-stale |

**authlib specifics:**
- `authlib.integrations.starlette_client` works directly in FastAPI (FastAPI is a Starlette subclass)
- `AsyncOAuth2Client` / `StarletteOAuth2App.authorize_redirect()` / `authorize_access_token()` handle PKCE automatically
- Reads IdP metadata from `server_metadata_url` (Dex's `/.well-known/openid-configuration`) — no hardcoded endpoints
- Requires `itsdangerous` + Starlette's `SessionMiddleware` to persist the PKCE state/nonce across the redirect

**Confidence:** HIGH — verified against authlib 1.6.x docs and multiple production examples

## 2. Session vs. Bearer Token Pattern

### Recommendation: Backend session cookies (Option A) — server-side OIDC flow

**Comparison:**

| Pattern | Security | DX complexity | Fits internal tool? |
|---|---|---|---|
| **(A) Backend sessions + httpOnly cookie** | HIGH — tokens never touch browser JS | Low backend complexity, zero frontend OIDC code | YES — ideal |
| **(B) Frontend PKCE + Bearer header** | MEDIUM — access token in memory (XSS risk) | Adds react-oidc-context, token refresh loop, AuthProvider wrapping | Overkill for internal |
| **(C) BFF proxy** | HIGH | Highest complexity — separate proxy service | Definitely overkill |

**Why Option A for KPI Light:**

1. This is a same-origin internal tool (frontend served at `/`, API at `/api`). No cross-origin complexity.
2. The OIDC authorization code flow + PKCE runs entirely on the FastAPI backend. The browser never sees an access token.
3. After the callback, FastAPI sets an `httpOnly; SameSite=Lax; Secure` session cookie containing only `{sub, email, name, id_token}`. All subsequent `/api/*` calls include the cookie automatically — no frontend Auth header logic.
4. CSRF risk with SameSite=Lax is negligible for a non-public tool with no cross-site embedding.
5. IETF 2025 guidance endorses BFF/server-side for SPAs; backend sessions are functionally equivalent for a same-origin SPA.

**Starlette SessionMiddleware behavior:** Uses `itsdangerous` to sign cookie data. Sets `HttpOnly=true` by default. Cookie payload is signed but NOT encrypted — do not store raw access/refresh tokens in it. Store only the minimal user identity claims.

## 3. Frontend Integration with wouter

### Recommendation: No OIDC library on the frontend (backend-session pattern eliminates the need)

Because auth state lives in an httpOnly cookie and a `/api/auth/me` endpoint exposes the current user, the frontend never needs to run PKCE or manage tokens.

**What the frontend actually needs:**

```
1. A `useCurrentUser` hook — TanStack Query query to GET /api/auth/me
   - Returns { sub, email, name } when logged in, null/401 when not
2. A redirect-to-login helper — window.location.href = "/api/auth/login"
   (not a wouter navigate — OIDC redirect must be a full navigation)
3. A <ProtectedRoute> wrapper — if currentUser is null, redirect to login
4. A logout button — POST /api/auth/logout (clears cookie) then hard redirect
```

**wouter-specific notes:**
- wouter has no built-in concept of auth-gated routes; this is standard pattern
- The `<ProtectedRoute>` is just a component that checks `useCurrentUser` and either renders children or calls `window.location.href = "/api/auth/login"`
- No react-oidc-context or oidc-client-ts needed — saves ~40 KB bundle

**If frontend PKCE were required** (not recommended), `react-oidc-context` 3.3.1 + `oidc-client-ts` 3.5.0 are the current standard. react-oidc-context wraps the whole app in `<AuthProvider>` and exposes `useAuth()`. It is router-agnostic and works with wouter, but it introduces a token-management complexity layer that the backend-session pattern avoids entirely. React 19 compatibility is not explicitly confirmed in docs but the library declares `react: ">=16"` as peer dep — likely works.

## 4. Auth-Less Dev Mode Toggle

### Pattern: Settings-based dependency factory with a no-op fallback

```python
# backend/app/config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    disable_auth: bool = False
    session_secret_key: str = "change-me-in-production"
    oidc_client_id: str = ""
    oidc_client_secret: str = ""
    oidc_server_metadata_url: str = ""

    class Config:
        env_file = ".env"

settings = Settings()
```

```python
# backend/app/auth/dependencies.py
from fastapi import Depends, HTTPException, Request
from app.config import settings

class CurrentUser:
    sub: str
    email: str
    name: str

    def __init__(self, sub: str, email: str, name: str):
        self.sub = sub
        self.email = email
        self.name = name

_DEV_USER = CurrentUser(sub="dev", email="dev@local", name="Dev User")

async def get_current_user(request: Request) -> CurrentUser:
    """Dependency — returns CurrentUser or raises 401.

    When DISABLE_AUTH=true (env), always returns a synthetic dev user.
    This means every endpoint works without a real session cookie.
    """
    if settings.disable_auth:
        return _DEV_USER

    user_data = request.session.get("user")
    if not user_data:
        raise HTTPException(status_code=401, detail="Not authenticated")
    return CurrentUser(**user_data)
```

**Usage on endpoints:**

```python
# Any router file
from fastapi import Depends
from app.auth.dependencies import get_current_user, CurrentUser

@router.post("/upload")
async def upload_file(
    file: UploadFile,
    db: AsyncSession = Depends(get_async_db_session),
    current_user: CurrentUser = Depends(get_current_user),  # add this line
) -> UploadResponse:
    ...
```

**Key properties of this pattern:**
- A single env var controls the entire auth path
- `DISABLE_AUTH=true` is safe to commit to `.env.example` with a clear comment
- No conditional imports or if/else scattered across routers
- FastAPI's `app.dependency_overrides[get_current_user]` can replace it in tests without any env var

**`.env` additions:**
```
DISABLE_AUTH=true          # set to false in production
SESSION_SECRET_KEY=<random 32-byte hex>
OIDC_CLIENT_ID=kpi-light
OIDC_CLIENT_SECRET=<from-dex-config>
OIDC_SERVER_METADATA_URL=https://dex.internal/dex/.well-known/openid-configuration
```

## 5. User Identity Persistence

### Minimal SQLAlchemy model

```python
# backend/app/models.py — add to existing file

class AppUser(Base):
    __tablename__ = "app_users"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    sub: Mapped[str] = mapped_column(String(255), nullable=False, unique=True)
    email: Mapped[str] = mapped_column(String(255), nullable=False)
    name: Mapped[str | None] = mapped_column(String(255), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), nullable=False, server_default=func.now()
    )
    last_seen_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), nullable=False, server_default=func.now()
    )
```

**Upsert on every successful OIDC callback:**

```python
# On callback, after token validation
await db.execute(
    pg_insert(AppUser)
    .values(sub=sub, email=email, name=name, last_seen_at=datetime.now(timezone.utc))
    .on_conflict_do_update(
        index_elements=["sub"],
        set_={"email": email, "name": name, "last_seen_at": datetime.now(timezone.utc)},
    )
)
await db.commit()
```

**Future upload attribution:** Add `uploaded_by_sub: Mapped[str | None] = mapped_column(String(255), nullable=True)` to `UploadBatch` — a soft FK (no FK constraint) avoids cascade issues if the user table is ever purged.

## 6. Auth Endpoints — Code Sketch

### Dependencies to add

```
# requirements.txt additions
authlib>=1.6.0
itsdangerous>=2.2.0
```

Starlette ships with FastAPI; SessionMiddleware is built-in.

### `backend/app/auth/router.py`

```python
from fastapi import APIRouter, Depends, Request
from fastapi.responses import RedirectResponse
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy.dialects.postgresql import insert as pg_insert
from datetime import datetime, timezone

from app.config import settings
from app.database import get_async_db_session
from app.models import AppUser
from app.auth.client import oauth  # see below

router = APIRouter(prefix="/api/auth")


@router.get("/login")
async def login(request: Request) -> RedirectResponse:
    """Kick off authorization code + PKCE flow."""
    redirect_uri = request.url_for("callback")  # -> /api/auth/callback
    return await oauth.dex.authorize_redirect(request, redirect_uri)


@router.get("/callback")
async def callback(
    request: Request,
    db: AsyncSession = Depends(get_async_db_session),
) -> RedirectResponse:
    """Receive code, exchange for tokens, upsert user, set session cookie."""
    token = await oauth.dex.authorize_access_token(request)
    userinfo = token.get("userinfo") or await oauth.dex.userinfo(token=token)

    sub = userinfo["sub"]
    email = userinfo.get("email", "")
    name = userinfo.get("name") or userinfo.get("preferred_username", "")

    # Upsert into app_users
    await db.execute(
        pg_insert(AppUser)
        .values(sub=sub, email=email, name=name, last_seen_at=datetime.now(timezone.utc))
        .on_conflict_do_update(
            index_elements=["sub"],
            set_={"email": email, "name": name, "last_seen_at": datetime.now(timezone.utc)},
        )
    )
    await db.commit()

    # Store minimal identity in signed session cookie (NOT the access/refresh token)
    request.session["user"] = {"sub": sub, "email": email, "name": name}

    return RedirectResponse(url="/")


@router.get("/me")
async def me(request: Request):
    """Frontend polls this to check session state. 401 = not logged in."""
    user = request.session.get("user")
    if not user:
        from fastapi import HTTPException
        raise HTTPException(status_code=401, detail="Not authenticated")
    return user


@router.post("/logout")
async def logout(request: Request) -> RedirectResponse:
    """Clear local session. Optionally redirect to Dex end_session_endpoint."""
    id_token_hint = request.session.get("id_token_hint")
    request.session.clear()

    # Attempt RP-initiated logout if Dex exposes end_session_endpoint
    # (see section 7 — Dex may not advertise this; check at runtime)
    metadata = await oauth.dex.load_server_metadata()
    end_session_url = metadata.get("end_session_endpoint")
    if end_session_url and id_token_hint:
        from urllib.parse import urlencode
        params = urlencode({
            "id_token_hint": id_token_hint,
            "post_logout_redirect_uri": "http://kpi-light.internal/",
        })
        return RedirectResponse(url=f"{end_session_url}?{params}")

    return RedirectResponse(url="/")
```

### `backend/app/auth/client.py`

## 7. Frontend Changes

### `useCurrentUser` hook

```typescript
// frontend/src/hooks/useCurrentUser.ts
import { useQuery } from "@tanstack/react-query";

export interface CurrentUser {
  sub: string;
  email: string;
  name: string;
}

export function useCurrentUser() {
  return useQuery<CurrentUser | null>({
    queryKey: ["currentUser"],
    queryFn: async () => {
      const res = await fetch("/api/auth/me");
      if (res.status === 401) return null;
      if (!res.ok) throw new Error("auth check failed");
      return res.json();
    },
    staleTime: 60_000,
    retry: false,
  });
}
```

### `ProtectedRoute` component

```typescript
// frontend/src/components/ProtectedRoute.tsx
import { useCurrentUser } from "@/hooks/useCurrentUser";

export function ProtectedRoute({ children }: { children: React.ReactNode }) {
  const { data: user, isLoading } = useCurrentUser();

  if (isLoading) return null; // or a spinner

  if (!user) {
    window.location.href = "/api/auth/login";
    return null;
  }

  return <>{children}</>;
}
```

## 8. Logout Flow — Dex Reality Check

**Current Dex status (as of April 2026):**
- Dex GitHub issue #1697 (opened April 2020) requests `end_session_endpoint` support
- The feature has NOT been merged into mainline Dex as of the research date
- Dex's `/.well-known/openid-configuration` does NOT advertise `end_session_endpoint`
- Confidence: MEDIUM (based on issue tracker + discovery doc pattern research; verify by hitting your Dex instance's `/.well-known/openid-configuration`)

**Consequence:** Full cross-application RP-initiated logout ("log out of KPI Light AND the wiki simultaneously") is not achievable via standard OIDC with Dex.

**Practical logout strategy for v1.11:**

```
Tier 1 (implement now): Local session logout
  POST /api/auth/logout → clears httpOnly session cookie → redirect to /
  User is logged out of KPI Light. Other apps are unaffected.

Tier 2 (implement when/if Dex adds end_session_endpoint):
  The /api/auth/logout handler already checks metadata.end_session_endpoint
  (see code sketch above) — it will activate automatically when Dex supports it.
  No frontend changes needed.

Tier 3 (workaround if cross-app logout is urgent before Dex supports it):
  Coordinate a shared logout page at a known URL that all apps redirect to,
  which clears each app's session in sequence.
  This is a non-standard workaround and introduces tight coupling between apps.
```

**Recommendation:** Ship Tier 1 for v1.11. Document that SSO session at Dex level will remain until token expiry. Set Dex token TTL (e.g., 1 hour) to bound the window.

## 11. Critical Pitfalls

**Pitfall 1: SessionMiddleware cookie size limit**
Starlette's SessionMiddleware encodes the entire session dict into a single cookie (4096-byte browser limit). Never store access tokens or refresh tokens in the session — only minimal claims (`sub`, `email`, `name`). Store `id_token_hint` only if needed for logout and only for the duration of the session.

**Pitfall 2: `https_only=True` breaks local HTTP dev**
`SessionMiddleware(https_only=True)` means the cookie is only sent over HTTPS. In local dev via HTTP (`localhost:8000`), the cookie will silently never arrive. Solution: set `HTTPS_ONLY=false` in `.env` for dev and `true` in production. Wire `settings.https_only` into the middleware config.

**Pitfall 3: authlib Starlette client requires SessionMiddleware BEFORE any router**
`authorize_redirect()` writes PKCE state to `request.session`. If `SessionMiddleware` is added after routers or via `app.mount()`, `request.session` will be absent and the redirect will fail with a `KeyError`. Always add middleware at the top of `main.py` before `include_router()` calls.

**Pitfall 4: Dex `server_metadata_url` path**
Dex typically serves OIDC discovery at `https://dex.host/dex/.well-known/openid-configuration` (note the `/dex/` path prefix, set by Dex's `issuer` config). Confirm the exact URL from the Dex config. A wrong `server_metadata_url` produces a generic `httpx` network error with no hint about the cause.

**Pitfall 5: `DISABLE_AUTH=true` in production**
If the env var is accidentally left as `true` in a Docker Compose production deploy, every request gets the synthetic dev user. Add a startup assertion:

```python
# in lifespan or main.py startup
import os
if os.getenv("DISABLE_AUTH", "false").lower() == "true":
    import warnings
    warnings.warn("DISABLE_AUTH is enabled — all requests authenticated as dev user")
```

**Pitfall 6: wouter `navigate()` vs `window.location.href` for OIDC redirect**
`navigate("/api/auth/login")` is a client-side route change — it will NOT trigger the FastAPI endpoint and will result in wouter's 404 fallback. The login redirect and logout form submission MUST use full page navigations (`window.location.href` or an HTML `<form method="POST">`).
