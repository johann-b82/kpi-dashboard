## Spacing Scale

Standard 8-point scale. All new signage components use these values only.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, chip internal padding |
| sm | 8px | `gap-2`, `p-2` | Inline padding, compact chip gap |
| md | 16px | `gap-4`, `p-4` | Default element / form field spacing |
| lg | 24px | `gap-6`, `p-6` | Card content padding, section spacing |
| xl | 32px | `gap-8`, `p-8` | Between major sections |
| 2xl | 48px | `gap-12` | Page header to first content block |
| 3xl | 64px | `pb-16` | Page bottom padding (action bar clearance) |

Exceptions:
- Launcher tile: fixed `w-[120px] h-[120px]` — mirrors existing tile dimensions exactly (LauncherPage.tsx pattern).
- Touch targets (drag handles, chip remove buttons): minimum 44px in either dimension via padding expansion.
- Preview dock split at `≥lg` breakpoint: left pane `w-2/5`, right pane `w-3/5` (fluid, not fixed).

## Typography

Inherited from the project's Geist Variable font. Phase 46 declares exactly 4 type sizes and 2 weights.

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|---------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` |
| Heading (card/section) | 20px | 600 (semibold) | 1.3 | `text-xl font-semibold` |
| Page title | 30px | 600 (semibold) | 1.2 | `text-3xl font-semibold` |

Notes:
- Status pill / badge text: `text-xs` (12px, weight 400) — same as existing `<Badge>` primitive usage.
- **One-off component exception — PairPage pairing code display:** `font-mono text-4xl font-semibold tracking-widest text-center w-full`. The 36px monospace size is used exclusively for the 6-digit pairing-code input on PairPage and is not part of the general type scale. It is not declared in the scale table above and must not be reused elsewhere.
- `font-medium` (`weight 500`) is not used in Phase 46. Any existing or drafted use of `font-medium` in signage components must be changed to `font-semibold`.
- All sizes sourced from existing project usage; confirmed in `SettingsPage.tsx`, `LauncherPage.tsx`, `DeleteConfirmDialog.tsx`.

**Primary visual anchor: tile grid**

- Navigation: `setLocation("/signage")` → server-side redirect to `/signage/media`

### 2. Signage Shell — Sub-Nav Button Group (SGN-ADM-03, D-04)

- Rendered by `SignagePage.tsx` above the active sub-page
- Three buttons: Media / Playlists / Devices (left-aligned, horizontal, grouped)
- Active state: `bg-primary text-primary-foreground` — filled button look
- Inactive state: `bg-transparent border border-border text-foreground hover:bg-muted`
- Button group gap: 0 (border-collapse style — no gap between adjacent buttons, shared borders)
- Routing: clicking a button calls `setLocation("/signage/media")` etc. — URL-driven active state derived from `useLocation()`
- NOT shadcn `<Tabs>` component — this is a custom button group with URL routing (D-04)
- Page title: `text-3xl font-semibold` — "Digital Signage" (i18n key `signage.admin.page_title`)

### 3. Media Library (SGN-ADM-04, D-01, D-02, D-03)

**Primary visual anchor: upload dropzone above the grid**

**Layout:** Card-based grid. Thumbnails at top of card, metadata below.

- Grid: `grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-4`
- Thumbnail area: fixed `h-32` with `object-fit: cover` for image/video; placeholder icon for URL/HTML/PPTX
- Metadata row: title (truncated, `truncate`), kind badge, size text (`text-xs text-muted-foreground`)
- Tags: rendered as small chips below metadata (non-interactive in list view)
- Actions per card: Edit (pencil icon), Delete (trash icon) — icon buttons, 36px touch target

**Upload area** (top of MediaPage, above grid):

- Single `<MediaUploadDropZone>` spanning full width
- Accepts: image/*, video/*, application/pdf, application/vnd.openxmlformats-officedocument.presentationml.presentation
- Idle: `bg-muted border-2 border-dashed border-border min-h-[120px]`
- Drag active: `bg-primary/5 border-solid border-primary`
- Processing: spinner (`<Loader2 animate-spin>`) + `text-muted-foreground` text
- PPTX routing: if `file.name.endsWith('.pptx')` → `POST /api/signage/media/pptx`; else → Directus upload + register
- Below dropzone: "Register URL / HTML" `<Button variant="outline" size="sm">` → opens a small `<Dialog>` form

**PPTX status pill** on media card:
- Auto-polls (`refetchInterval` 3s) until `done` or `failed` terminal state
- `pending`: muted grey badge
- `processing`: amber badge with `animate-pulse`
- `done`: green badge (pill disappears after 5s or on user interaction)
- `failed`: red badge + error tooltip showing `conversion_error` text on hover

**In-use delete dialog** (`<MediaDeleteDialog>`):
- Triggered when backend returns 409 on `DELETE /api/signage/media/{id}`
- Title: "Cannot delete — in use" (i18n: `signage.admin.media.delete_in_use_title`)
- Body: "This media is used by {{count}} playlist(s). Remove it from all playlists first." (i18n: `signage.admin.media.delete_in_use_body`)
- Single action: "Close" (outline button) — no destructive confirm (DELETE RESTRICT prevents it server-side)
- Count derived from `playlist_ids.length` extracted from the 409 response body

### 9. PlayerRenderer (D-09, D-10, SGN-DIFF-02)

**Props interface:**
```tsx
interface PlayerRendererProps {
  items: PlaylistItem[]; // in-memory form state or server data
  className?: string;    // injected by parent for sizing
}
```

**Internal behavior:**
- Auto-advances through items based on `item.duration_s` (1s precision timer via `useEffect` + `setInterval`)
- `transition="fade"` items: CSS `opacity` transition 300ms between item swaps
- `transition="cut"` items: immediate swap (no transition)
- Loops back to item 0 after last item
- Pause/resume: none required for admin preview (D-10)
- Current item index held in `useState`; reset to 0 when `items` prop reference changes (playlist save)

**Format dispatch logic:**
```
item.kind === "image"  → <ImagePlayer uri={...} />
item.kind === "video"  → <VideoPlayer uri={...} />
item.kind === "pdf"    → <PdfPlayer uri={...} autoFlipSeconds={item.duration_s} />
item.kind === "url"    → <IframePlayer src={...} />
item.kind === "html"   → <HtmlPlayer html={...} />
item.kind === "pptx"   → <PptxPlayer slidePaths={item.slide_paths} />
```

**Format handler specs:**

| Handler | Implementation | Notes |
|---------|---------------|-------|
| ImagePlayer | `<img src={uri} className="w-full h-full object-contain" alt="">` | `opacity` CSS transition on item swap |
| VideoPlayer | `<video src={uri} muted autoPlay playsInline loop className="w-full h-full object-contain">` | `loop` for admin preview only; Phase 47 removes loop |
| PdfPlayer | `react-pdf <Document>` + `<Page>` with `renderTextLayer={false}` `renderAnnotationLayer={false}` | Auto-page-flip via `setInterval(autoFlipSeconds * 1000)`; use `pdfjs.version` for worker URL if needed |
| IframePlayer | `<iframe src={uri} sandbox="allow-scripts allow-same-origin" className="w-full h-full border-0">` | No `allow-forms` or `allow-popups` |
| HtmlPlayer | `<iframe srcDoc={html} sandbox="allow-scripts" className="w-full h-full border-0">` | Backend pre-sanitizes via nh3; frontend renders trusted content in sandbox |
| PptxPlayer | `<img>` cycle through `slide_paths` array; advance every `duration_s / numSlides` seconds | Falls back to first slide if `slide_paths` empty |

## Data Fetching Contract

All signage queries use the `signageKeys` factory (defined in `frontend/src/lib/queryKeys.ts`):

```ts
signageKeys.media()          → ["signage", "media"]
signageKeys.mediaItem(id)    → ["signage", "media", id]
signageKeys.playlists()      → ["signage", "playlists"]
signageKeys.playlistItem(id) → ["signage", "playlists", id]
signageKeys.devices()        → ["signage", "devices"]
signageKeys.tags()           → ["signage", "tags"]
```

**Refetch intervals by surface:**

| Surface | refetchInterval | Notes |
|---------|----------------|-------|
| Media list | none (mutation-invalidation only) | `refetchOnWindowFocus: true` default |
| PPTX media item (converting) | 3s | Stop when `done` or `failed` |
| Playlists list | none | Mutation-invalidation |
| Playlist editor | none | Form state owns the data after load |
| Devices table | 30s | Live status chip updates |

**Mutation pattern** (all signage mutations):
1. Call `apiClient<T>()` via `useMutation`
2. On success: `queryClient.invalidateQueries({ queryKey: [...] })` + `toast.success(t("..."))`

## Accessibility Contract

- All icon-only buttons: `aria-label` required (i18n key)
- Drag handles: `aria-roledescription="drag handle"` + `aria-label="Drag to reorder {{title}}"` (via @dnd-kit `attributes`)
- Status chips: `<span>` with visible text — not icon-only (screen readers read the text)
- TagPicker: combobox ARIA pattern — `role="combobox"` on input, `role="listbox"` on dropdown, `aria-activedescendant` tracks highlighted option
- Keyboard navigation: all interactive elements reachable via Tab; drag-reorder via Space+ArrowUp/Down via @dnd-kit KeyboardSensor
- Form validation errors: `aria-describedby` linking input to error `<p>` element
- Dialog focus trap: provided by Radix UI `<Dialog>` primitive (no additional setup needed)
