## Spacing Scale

Declared values (multiples of 4, Tailwind default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps inside buttons, inline chip padding |
| sm | 8px (`gap-2`, `p-2`) | KPI card label-to-value gap, toggle group inner padding |
| md | 16px (`gap-4`, `p-4`) | Default element spacing, card padding, form row gap |
| lg | 24px (`gap-6`, `p-6`) | KPI card internal padding, section padding |
| xl | 32px (`gap-8`, `p-8`) | Dashboard grid gap (cards ↔ chart), page top padding |
| 2xl | 48px (`gap-12`, `py-12`) | Major section breaks (nav bar ↔ page content) |
| 3xl | 64px (`py-16`) | Empty/error state vertical centering offset |

Layout rules:
- Dashboard page max width: `max-w-7xl` (80rem / 1280px), centered with `mx-auto px-6`
- Top nav bar height: 64px (`h-16`) — fixed across both Dashboard and Upload pages
- KPI card grid: 3 columns on `lg:` (≥1024px), 1 column below — no tablet portrait layout (v1 is desktop-only per INFR-03)
- Chart card: full width, min-height 400px (`min-h-[400px]`)
- Date-range control row: 16px gap between preset buttons and calendar picker trigger

Exceptions: none. Touch targets are not a concern — desktop-only per Requirements OUT OF SCOPE.

## Typography

All values use Tailwind v4 utility classes that resolve from the `base-nova` preset. Weights limited to **two**: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Body | 14px | 400 | 1.5 (`leading-6`) | `text-sm` | Default page text, table cells, form labels, nav links |
| Label | 12px | 600 | 1.33 (`leading-4`) | `text-xs font-semibold uppercase tracking-wide` | KPI card label ("Total Revenue"), chart axis labels, filter labels |
| Heading | 20px | 600 | 1.4 (`leading-7`) | `text-xl font-semibold` | Section titles ("Revenue over time"), dialog titles |
| Display | 32px | 600 | 1.2 (`leading-9`) | `text-3xl font-semibold tabular-nums` | KPI card values (e.g. "€1,234,567.89") — `tabular-nums` is REQUIRED so metric card digits align vertically |

Formatting rules:
- All monetary values: localized formatting via `Intl.NumberFormat(locale, { style: 'currency', currency: 'EUR' })` — locale follows `i18n.language` (`de-DE` / `en-US`)
- All counts: `Intl.NumberFormat(locale).format(n)` — no raw `toString()`
- Timestamps in "last updated" indicator: relative when < 24h (`vor 5 Minuten` / `5 minutes ago`) via a tiny local formatter, absolute otherwise (`11.04.2026 14:23` / `Apr 11, 2026 2:23 PM`)
- No font family overrides — Geist Variable ONLY

## Color

60/30/10 split:

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` (white) | `--background` / `bg-background` | Page background, chart canvas background |
| Secondary (30%) | `oklch(0.97 0 0)` (near-white neutral) | `--muted` / `bg-muted`, `--card` for cards | KPI card surfaces, chart card, nav bar background, date picker popover |
| Accent (10%) | `#2563eb` (blue-600) | `--color-primary` | RESERVED — see list below |
| Semantic success | `#16a34a` (green-600) | `--color-success` | Revenue chart series stroke + KPI "Total Revenue" value tint (OPTIONAL — may stay neutral if checker prefers a single accent) |
| Destructive | `#dc2626` (red-600) | `--color-destructive` | Error toasts, API failure banners only |
| Chart neutrals | `--chart-1` through `--chart-5` (OKLCH grayscale) | existing | Chart axes, gridlines, bar chart bars (when in bar mode) |

**Accent (`--color-primary` blue) is reserved for:**
1. Active state of granularity toggle (`daily` / `weekly` / `monthly`) — selected button background
2. Active state of chart type toggle (line / bar) — selected button background
3. Active state of date-range preset button (`This month` / `This quarter` / `This year` / `All time`)
4. Navigation bar active-link underline indicator (Dashboard vs Upload)
5. Calendar date picker — selected day cell, range endpoints, hover ring
6. Focus ring (`ring-2 ring-primary`) on all interactive elements
7. Primary CTA button background (none in Phase 3 — dashboard has no form submissions)

**Accent is NOT used for:**
- KPI card borders or backgrounds (they stay neutral `bg-card`)
- Chart line stroke (use `--color-success` green for revenue — semantic match)
- Nav bar background (stays `bg-card`/neutral)
- Body text links (none in Phase 3)

**Revenue series color decision (RESOLVED — Claude's discretion per D-44 not present; resolving here):**
- Line chart stroke and bar chart fill: `--color-success` (`#16a34a`, green-600). Rationale: green is the universal financial positive signal; the checker's "60/30/10 + reserved accent" rule is preserved because blue remains the UI control accent and green appears ONLY in the chart, not in general UI. The two colors never compete on the same element.
- Axes, gridlines, tick labels: `--muted-foreground` (`oklch(0.556 0 0)`, neutral gray)
- Tooltip background: `bg-popover` with `border-border`

Contrast targets (WCAG AA):
- Body text (`--foreground` on `--background`): 19:1 — PASS
- KPI display values: same — PASS
- Active toggle button (white on `#2563eb`): 8.6:1 — PASS
- Chart line (`#16a34a` on white): 3.9:1 — PASS for non-text graphical objects (WCAG 1.4.11, requires ≥3:1)

## Responsive Strategy (INFR-03)

Requirement INFR-03 says "usable on 1080p+ desktop screens." Out-of-scope explicitly excludes mobile. Strategy:

- Single breakpoint that matters: `lg:` (≥1024px). The dashboard is designed and tested at **1920×1080** as the primary target and remains functional down to 1280px wide.
- Below `lg:` (rare case — resized window), KPI cards collapse from `grid-cols-3` to `grid-cols-1` and the chart stays full-width. No hamburger menu, no mobile nav — the top nav bar stays horizontal.
- Max content width `max-w-7xl` (1280px) — on 1920px screens the content is centered with generous white-space margins (this is intentional, not a bug).

## Interaction Contracts

1. **Initial load**: Default date range = current calendar year (D-13). Default granularity = monthly. Default chart type = line (D-09). KPI endpoints called once with these params.
2. **Changing a preset range** instantly refetches KPIs and chart (TanStack Query `useQuery` with the range as part of the queryKey — automatic).
3. **Opening the custom picker**: popover opens below the "Custom…" button. User picks `from`, then `to`. "Apply range" button commits the change. "Reset" clears back to default. Closing without "Apply" discards.
4. **Changing granularity**: refetches chart only (cards are granularity-independent).
5. **Changing chart type**: client-side swap, no refetch.
6. **After upload (Phase 2 flow)**: upload mutation `onSuccess` calls `queryClient.invalidateQueries({ queryKey: ['kpis'] })` AND `['latest-upload']`. Dashboard auto-updates next time it renders or immediately if mounted (CF-02).
7. **Focus management**: Every interactive element gets `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2`. Tab order = nav bar → date filter → granularity → chart type → (chart is non-interactive for v1).
8. **Freshness indicator** refetch: once on mount, plus after upload invalidation. No polling.
