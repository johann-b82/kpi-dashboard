## Component Inventory

### LauncherPage grid

- Container: CSS grid, `grid-template-columns: repeat(auto-fill, minmax(120px, 1fr))`, `gap: 24px` (lg token)
- Max tile width per column: 120px; grid scales to 4 columns on desktop (≥640px), 3 on tablet (≥480px), 2 on mobile
- Page heading: `settings.app_name` from `useSettings()` — `text-2xl font-semibold` — source: CONTEXT.md D-11

### Tile card (active)

- Size: 120×120px, square, `rounded-2xl` (matches existing shadcn token `--radius-2xl`)
- Background: `bg-card` token
- Border: `border border-border` (1px, uses existing token)
- Hover: `hover:bg-accent/10 transition-colors` — consistent with NavBar icon hover pattern
- Cursor: `cursor-pointer`
- Click: navigates to target route via wouter `useLocation`
- Layout: flex column, icon centered, label below — `flex flex-col items-center justify-center gap-2 p-4`
- Icon size: 40px (`w-10 h-10`) — 1/3 of tile height, leaves adequate label space
- Label: 12px, weight 500, `text-muted-foreground`, centered, single line (truncate if needed)

### Tile card (coming-soon / greyed)

- Same card shell as active tile
- Opacity: `opacity-40` (40% — lower end of D-04 range for stronger visual distinction)
- No hover effect: `pointer-events-none`
- Cursor: default (inherited, no change)
- No overlay, no tooltip, no badge — source: CONTEXT.md D-04

### Tile icons

- Active KPI Dashboard tile: `LayoutDashboard` from lucide-react — source: CONTEXT.md D-06
- Placeholder tiles: `Box` icon from lucide-react — best visual balance at 120px tile size (simple, neutral, unambiguous placeholder) — source: CONTEXT.md D-07 (Claude's discretion)

### Auth wiring (no visual surface)

- `AuthGate.tsx`: `setLocation("/home")` on successful login — source: CONTEXT.md D-08
- `/home` Route in `App.tsx` `<Switch>` — source: CONTEXT.md D-09
- No visual change to login form or NavBar

### SubHeader on /home

- Renders in default/empty state — no route-specific content — source: CONTEXT.md D-03
- SubHeader component renders as-is with no props; its default empty render is the correct output

## Copywriting Contract

All copy is i18n-keyed via `react-i18next` in `en.json` / `de.json`. Keys use `launcher.*` namespace.

| Element | i18n Key | EN Copy | DE Copy |

| Primary CTA (tile click) | n/a — navigation, no label | — | — |
| Empty state | not applicable — grid always shows 4 tiles | — | — |
| Error state | not applicable — no data fetching in this phase | — | — |
| Destructive confirmation | not applicable — no destructive actions | — | — |

Note: No empty state, error state, or destructive confirmation surface exists in this phase. The grid always renders with exactly 4 tiles (1 active + 3 coming-soon).
