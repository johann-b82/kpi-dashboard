## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**SubHeader internal spacing:** `gap-3` (12px) between left-slot items and between right-slot items — matches existing `SubHeader.tsx` pattern confirmed at line 103 and 124.

**SubHeader chrome:** fixed `h-12` (48px) at `top-16`. Total chrome height (NavBar h-16 + SubHeader h-12 = 112px) is unchanged by this phase.

**SensorsPage body top padding:** `pt-4` (16px). Whether to drop it is left to Claude's discretion (CONTEXT.md §Discretion). Default: preserve it to match other dashboard routes.

Exceptions: SubHeader height is `h-12` (not a multiple of 4 steps on the scale above, but an established app chrome token — do not change).

### Button contract for PollNowButton in SubHeader (D-07)

- Variant: `default` (primary fill)
- Size: `sm` → `h-7` per `button.tsx` size scale. Note: CONTEXT.md D-07 says "h-8" but references "size=sm". The `sm` size in the Phase 55 Button primitive resolves to `h-7`. The `default` size resolves to `h-8`. Use `size="sm"` as stated in D-07; the executor must verify the visual result matches the SubHeader `h-12` proportions.
- Icon: `RefreshCw` from lucide-react (idle state) — matches the app's existing "kick a refresh job" convention (`PersonioCard.tsx` line 214). Left of label.
- Loading icon: `Loader2` with `animate-spin` (D-08) — swaps in place of `RefreshCw` when `mutation.isPending === true`.
- Label: `t("sensors.poll.now")` — "Poll now" / "Jetzt messen" — existing i18n key, no new key needed.
- Disabled: `true` when `mutation.isPending` (existing behavior preserved).
- `aria-busy`: `{mutation.isPending}` (existing attribute preserved).

### SegmentedControl contract for SensorTimeWindowPicker in SubHeader (D-03)

- Component: existing `SegmentedControl` (not Toggle — has 5 options, not 2).
- Options: `["1h", "6h", "24h", "7d", "30d"]` via `SENSOR_WINDOWS` constant.
- Labels: `t("sensors.window.{w}")` — existing keys.
- `aria-label`: `t("sensors.window.aria")` — existing key ("Time window").
- Placement: left slot, same `gap-3` flex row as Sales/HR `Toggle` on `/sales|/hr`.
- No vertical divider between picker and left-slot edge (matches `DateRangeFilter` on `/sales` which has no divider — confirmed in `SubHeader.tsx`).

### SensorFreshnessIndicator (right slot — unchanged)

- Stays at its existing position in the right slot.
- PollNowButton inserts to its left (right-slot order: `PollNowButton` → `SensorFreshnessIndicator`), per D-04.

## Interaction States

### PollNowButton — idle

- Icon: `RefreshCw` size-3.5 (sm button svg default), left of label.
- Label: "Poll now" / "Jetzt messen".
- Variant: default (primary fill).
- Pointer: enabled.

### PollNowButton — polling (mutation.isPending)

- Icon swaps to `Loader2 animate-spin`.
- `disabled={true}` → `pointer-events-none opacity-50` (Button primitive handles this).
- `aria-busy="true"`.
- No inline status text in the right slot (D-08 — keeps slot uncluttered).

### SensorTimeWindowPicker — active segment

- Active segment background: `--primary` (or token-driven SegmentedControl active style already established by Phase 54).
- No change to SegmentedControl visual — reuse existing.

## Accessibility Contract

- `PollNowButton` accessible name: visible label "Poll now" / "Jetzt messen" (icon + label pattern, D-07).
- `SensorTimeWindowPicker` accessible name: `aria-label={t("sensors.window.aria")}` on the SegmentedControl (existing).
- Focus ring: inherited from `Button` primitive (`focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50`) and SegmentedControl — no new focus ring work needed.
- `aria-busy` on PollNowButton: existing attribute preserved.
- Keyboard: PollNowButton is a `<ButtonPrimitive>` (base-ui) — Enter/Space activates. SegmentedControl keyboard nav is established by Phase 54.
- Reduced motion: SegmentedControl indicator and `animate-spin` on `Loader2` respect `prefers-reduced-motion` per CSS transitions. No additional work needed in this phase (Phase 59 owns the sweep).
