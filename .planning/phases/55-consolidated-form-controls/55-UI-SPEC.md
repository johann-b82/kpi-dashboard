## Spacing Scale

Declared values (all multiples of 4; resolved from Tailwind default scale — no custom spacing tokens introduced this phase):

| Token | Value | Usage in Phase 55 |
|-------|-------|-------------------|
| xs | 4px (`gap-1`, `p-1`) | Popup inner padding (Select/Dropdown `Popup`), icon-label gaps inside items |
| sm | 8px (`p-2`, `gap-2`, `px-2`) | Popup item horizontal padding, `gap-2` between icon + label inside `DropdownItem` |
| md-tight | 10px (`px-2.5`) | Horizontal padding inside form-control surfaces (`Input`, `Select` trigger, `Textarea`) — matches existing `Input` |
| md | 16px (`gap-4`, `p-4`) | Default spacing between labelled fields in consumer layouts (not enforced by primitives) |
| lg | 24px (`gap-6`, `p-6`) | Section padding in consumer layouts (not enforced by primitives) |

**Phase 55 is spacing-neutral at the layout level.** Internal padding is fixed:

| Primitive | Horizontal pad | Vertical pad | Height |
|-----------|---------------|--------------|--------|
| `Input` (existing) | `px-2.5` (10px) | `py-1` (4px) | `h-8` (32px) |
| `Select` trigger (new) | `px-2.5` (10px) | `py-1` (4px) | `h-8` (32px) — MUST match Input |
| `Button` default (existing) | `px-3` (12px) | (auto) | `h-8` (32px) |
| `Button` xs / sm | `px-2` / `px-2.5` | (auto) | `h-6` / `h-7` |
| `Textarea` (new) | `px-2.5` (10px) | `py-1.5` (6px) | `min-h-16` (≈3 rows, `resize-y`) — CTRL-03 exempt (inherently multi-line) |
| `Select` / `Dropdown` popup item | `px-2` (8px) | `py-1` (4px) | auto |
| `Select` / `Dropdown` popup container | `p-1` (4px) | `p-1` (4px) | auto; `sideOffset={4}` |

Exceptions (documented in-source):
- **File inputs** (`<input type="file">`) stay native per D-10 — spacing / height are browser-native; must be annotated with `// CTRL-02 exception: native file picker — …` at each call site (`DropZone.tsx`, `LogoUpload.tsx`, `MediaUploadDropZone.tsx`, plus any others surfaced during migration).
- **`Textarea`** intentionally does NOT use `h-8` — multi-line contract. Uses `min-h-16`.
- **`Button.xs` (`h-6` / 24px)** and **`Button.sm` (`h-7` / 28px)** allowed per D-05 for dense surfaces (table row actions, sub-toolbars). NOT standard. JSDoc above `buttonVariants` must say so.
- **`table.tsx` `<th className="h-10 …">`** is NOT a form control — out of scope for CTRL-03.
- **`SalesTable.tsx:68` and `EmployeeTable.tsx:85`** currently pass `className="pl-9 h-9"` on `<Input>` — the `h-9` override must be stripped during the Wave-2(c) `<input>` migration plan; keep `pl-9` (icon padding for the search-icon pattern).

## Typography

Phase 55 introduces NO new type styles. All primitives inherit the existing type ramp established by Geist Variable + `text-sm`/`text-base` responsive pair used by `Input`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (desktop) | 14px (`md:text-sm`) | 400 (regular) | 1.5 (default) |
| Body (mobile — form controls only) | 16px (`text-base`) | 400 | 1.5 — prevents iOS zoom-on-focus |
| Label | 14px (`text-sm`) | 500 (medium, inherited from existing `Label`) | 1.4 |
| Heading (consumer pages — unchanged this phase) | 20/28px | 600 | 1.2 |

Weight palette: regular (400) + medium (500) + semibold (600). No new weights introduced.

**Primitive-specific type rules:**

| Primitive | Text size class |
|-----------|----------------|
| `Input` / `Select` trigger / `Textarea` | `text-base md:text-sm` (16px mobile → 14px desktop) |
| `Button` default | `text-sm font-medium` (inherited from existing cva) |
| `Button.xs` / `Button.sm` | `text-xs` (12px) |
| `Select` / `Dropdown` popup item | `text-sm` (14px) |
| File-input exception (native) | browser-native |

No hardcoded font sizes permitted in call sites. If a consumer needs a different size, primitive `className` override is discouraged — file a design follow-up instead.

## Color

Phase 55 consumes ONLY existing tokens. No new color values are introduced.

| Role | Token | Light (`:root`) | Dark (`.dark`) | Usage in Phase 55 |
|------|-------|----------------|----------------|-------------------|
| Dominant (60%) | `--background` / `--foreground` | `oklch(1 0 0)` / `oklch(0.145 0 0)` | `oklch(0.145 0 0)` / `oklch(0.985 0 0)` | Page surface behind controls |
| Secondary (30%) | `--card` / `--muted` / `--popover` | white / near-white | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | Select/Dropdown popup surface (`bg-popover`), hover row (`data-[highlighted]:bg-muted`) |
| Accent (10%) | `--primary` / `--ring` | `oklch(0.205 0 0)` / `oklch(0.708 0 0)` | `oklch(0.922 0 0)` / `oklch(0.556 0 0)` | Focus ring on ALL form controls (`focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50`); solid button fill (`bg-primary`) |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` | `oklch(0.704 0.191 22.216)` | Invalid-state border + ring on ALL form controls; destructive button variant; destructive dropdown items (`className="text-destructive"`) |
| Border / Input surface | `--border` / `--input` | `oklch(0.922 0 0)` | `oklch(1 0 0 / 10%)` / `oklch(1 0 0 / 15%)` | `border border-input` on every control; `dark:bg-input/30` for field fill in dark mode |

### Accent reserved for (explicit list — CTRL-04 contract)

**`--ring` / `--primary` may appear ONLY on:**
1. Focus ring of every form-control primitive (`Button`, `Input`, `Select` trigger, `Textarea`, `Dropdown` trigger).
2. `Button variant="default"` fill (`bg-primary text-primary-foreground`).
3. Existing Toggle indicator (unchanged from Phase 54).

Never on: static text, informational icons, row separators, chart axes (chart colors use `--chart-*` tokens).

### Destructive reserved for (explicit list)

**`--destructive` may appear ONLY on:**
1. `aria-invalid` state on any primitive (border + ring at 20% alpha).
2. `Button variant="destructive"` (solid fill).
3. Text class `text-destructive` inside a `DropdownItem` for a destructive action row (`Delete`, `Revoke`, etc.).
4. Inline caller-rendered error messages: `<p className="text-sm text-destructive">`.

No hex literals. No `dark:` color literal overrides — milestone hazard #3. All invalid visuals flow through `aria-invalid:*` utilities; dark-mode variants already baked into the shared class chain (see §Interaction State Matrix).

## Interaction State Matrix (CTRL-04 core contract)

**This is the SSOT for CTRL-04 visual parity.** Every form-control primitive MUST apply this exact class chain (copied from `input.tsx`). If duplication grows unwieldy, planner extracts to `frontend/src/components/ui/_shared.ts` per D-08 discretion.

| State | Selector / trigger | Class fragment |
|-------|-------------------|----------------|
| Default surface | (base) | `border border-input bg-transparent dark:bg-input/30 rounded-lg` |
| Hover | (native — no explicit hover on Input/Select/Textarea; Button only) | Button: `hover:bg-*` per cva variant |
| Focus-visible | `:focus-visible` | `focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50 outline-none` |
| Disabled | `:disabled` / `[disabled]` | `disabled:pointer-events-none disabled:cursor-not-allowed disabled:opacity-50` (+ `disabled:bg-input/50 dark:disabled:bg-input/80` on Input/Textarea) |
| Invalid | `[aria-invalid="true"]` | `aria-invalid:border-destructive aria-invalid:ring-3 aria-invalid:ring-destructive/20 dark:aria-invalid:border-destructive/50 dark:aria-invalid:ring-destructive/40` |
| Placeholder | `::placeholder` | `placeholder:text-muted-foreground` |
| Popup item (Select, Dropdown) — highlighted | `[data-highlighted]` | `data-[highlighted]:bg-muted data-[highlighted]:text-foreground` |
| Popup item — disabled | `[data-disabled]` | `data-[disabled]:pointer-events-none data-[disabled]:opacity-50` |

### Shape tokens (shared across all primitives)

| Property | Value | Rationale |
|----------|-------|-----------|
| Radius | `rounded-lg` (`--radius-lg` = `0.625rem` = 10px) | Matches existing Input/Button |
| Border width | `1px` (`border`) | Matches existing Input |
| Focus ring width | `3px` (`ring-3`) | Matches existing Input; NOT `ring-2` — do not shrink |
| Popup shadow | `shadow-md ring-1 ring-foreground/10` | Matches existing `popover.tsx` |
| Popup offset | `sideOffset={4}` | Consistent with Popover defaults |
| Popup z-index | `z-50` | Consistent with existing Portals |

### Accessible-name contract

Every migrated control MUST expose an accessible name (milestone hazard #9, reinforced by A11Y-02 in Phase 59). For Phase 55 primitives:

- `Input` / `Textarea` / `Select` trigger: associate with `<Label htmlFor>` (or `aria-label` if visually label-less).
- `Button` icon-only variants (`icon`, `icon-xs`, `icon-sm`): REQUIRE `aria-label`.
- `Dropdown` trigger: REQUIRE `aria-label` (e.g. `"Actions"`) since the kebab icon has no text.
- `Select` trigger: `<SelectPrimitive.Value placeholder="…">` is a visual placeholder, not an accessible name — labeled field still required.

### Keyboard contract

- `Select`: typeahead; ArrowUp/Down through options; Enter selects; Escape closes; Home/End jump. (Provided by base-ui.)
- `Dropdown`: ArrowUp/Down through items; Enter/Space activates; Escape closes; focus returns to trigger. (Provided by base-ui.)
- `Textarea`: native — Tab INDENTS unless `e.preventDefault()` is called (we do not preventDefault; Tab moves focus, which is correct for forms).
- `Button`: native — Space/Enter activates.
- Milestone hazard #9: every focus ring must be visible in light AND dark.

### Reduced-motion contract

`prefers-reduced-motion` (milestone hazard #8) is ALREADY honored by `toggle.tsx` via `usePrefersReducedMotion`. Phase 55 primitives introduce NO animations beyond base-ui's built-in popup open/close transitions, which base-ui itself gates on user motion preference. No per-primitive action required. Spot-check: open DevTools → Emulate `prefers-reduced-motion: reduce` → confirm Select / Dropdown popups still open (no slide/fade required).

## Copywriting Contract

Phase 55 ships **primitives only** — no new user-facing copy enters the app this phase. Migration plans (Wave 2) REPLACE raw elements with primitives while preserving existing labels verbatim (DE + EN).

Milestone hazard #1 (DE/EN i18n parity) is NOT triggered by Phase 55 migrations because no new i18n keys are introduced. Any migration plan that accidentally adds, renames, or removes a key MUST update both `de.json` AND `en.json` with matching key count (enforced by CI parity script).

| Element | Copy / policy |
|---------|---------------|
| Primary CTA (no new CTAs this phase) | n/a — migrations preserve existing button labels verbatim |
| Empty state heading | n/a — primitives are stateless; empty-state copy stays in consumer pages unchanged |
| Empty state body | n/a |
| Error state (primitive-level visual) | No inline error text rendered by primitives (D-07). Callers own error-message rendering via `<p className="text-sm text-destructive">{t("errors.<key>")}</p>` beneath the field. |
| Destructive confirmation | n/a this phase — destructive-action confirm dialogs are Phase 57's concern (`DeleteDialog` standardization). `Dropdown` destructive rows render `className="text-destructive"` but the dialog wiring stays as-is. |
| Accessible names for icon-only controls | Every `<Button size="icon*">` and every `<DropdownTrigger>` MUST carry `aria-label` with a DE+EN-translated string (existing convention; no new keys). |
| Native file-input exception comment | EXACT copy (grep-verifiable): `// CTRL-02 exception: native file picker — primitive <Input> does not wrap file-type inputs (browser-native styling retained).` |

**Tone guardrails (inherited from milestone):** DE copy uses du-tone. EN copy uses plain declarative verb+noun. No marketing language in internal admin surfaces. These apply to downstream phases that touch copy; Phase 55 only inherits.
