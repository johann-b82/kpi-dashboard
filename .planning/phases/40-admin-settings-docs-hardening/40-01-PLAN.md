**Load:** combine `useQuery(['settings'])` (for globals ‚Äî `Settings` interface already exposes the five sensor_* fields) and `useQuery(['sensors', 'admin'], fetchSensors)` (fresh list ‚Äî do NOT reuse `sensorKeys` from Phase 39 dashboard to avoid interfering with the 15s refetchInterval).

**Save algorithm** (atomicity note below):
1. For every row where `_markedForDelete` + `id != null` ‚Üí `deleteSensor(id)`.
2. For every row where `id == null` + not marked deleted ‚Üí build SensorCreatePayload; community MUST be non-empty (validated pre-save via `validate()`); `createSensor(payload)`. Error if community blank on a new row.
3. For every row where `id != null` + not marked deleted + has any field differing from snapshot ‚Üí build SensorUpdatePayload that ONLY includes dirty fields; if `communityDirty === false` ‚Üí OMIT `community` (preserves stored ciphertext); `updateSensor(id, payload)`.
4. If any global changed ‚Üí `updateSettings(globalsPayload)` where the payload includes ONLY changed globals (compare each to snapshot; empty string threshold = skip).
5. On all-success: invalidate `['sensors']` + `['sensors', 'admin']` + `['settings']` + `sensorKeys.all` (imported from Phase 39 `queryKeys`). Refetch + rebuild snapshot.
6. On any failure: do NOT rollback already-committed rows (explicit design ‚Äî this is "best-effort partial apply"; the UI shows which rows succeeded via toast aggregation in 40-02). Document this inline as: "This is NOT transactional across rows; Phase 41 could batch via a /api/sensors/bulk endpoint if needed. Keeping it simple for MVP ‚Äî a failed row re-appears in the draft on next snapshot refresh."

**Dirty detection:** `isDirty` = `JSON.stringify(snapshot) !== JSON.stringify(currentDraft)` OR any row has `_markedForDelete: true` OR any row has `id === null` (new, unsaved).

- blank name ‚Üí `name_required`
- blank host ‚Üí `host_required`
- duplicate names across non-deleted rows ‚Üí `name_duplicate`
- non-positive temperature_scale / humidity_scale ‚Üí `positive_number`
- blank community on a new (id===null) row ‚Üí `community_required` (new i18n key to add in Task 3)
