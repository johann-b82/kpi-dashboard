# Phase 6 — UI Design Contract

> Visual and interaction contract for the Settings page and its sub-components. Generated by gsd-ui-researcher from CONTEXT.md + REQUIREMENTS.md + existing project tokens. Verified by gsd-ui-checker.

**Phase goal:** A full Settings page at `/settings` where users can edit the 6 semantic color tokens, upload a logo, edit the app name, see changes live, Save, Discard, and Reset — with an unsaved-changes guard. Covers SET-01, BRAND-05, BRAND-07, BRAND-08, UX-01, UX-02.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Gap between color swatch and hex input; icon-to-label gap inside buttons |
| sm | 8px | Gap between stacked label + input; gap between action-bar buttons |
| md | 16px | Card internal padding, gap between form rows, gap between logo thumbnail and dropzone |
| lg | 24px | `CardHeader`/`CardContent` separation; gap between Identity card and Colors card |
| xl | 32px | Page horizontal gutter on desktop; bottom padding reserved on scroll container so sticky action bar never covers content |
| 2xl | 48px | Vertical rhythm between page title and first card |
| 3xl | 64px | Not used in this phase |

**Exceptions:** none. The sticky action bar uses `md` (16px) internal padding on all sides.

**Responsive grid rules (BRAND-05 — 6 color pickers):**
- Mobile (`<md`): 1 column, each picker full-width, `gap-md` (16px) between rows
- Desktop (`md` and up): 2 columns, `gap-md` (16px) horizontal + vertical

**Logo thumbnail:** fixed 120×120 px, `object-contain`, `rounded-md` — independent of spacing scale (it is a content asset, not layout).

## Typography

Project uses one font family only: **Geist Variable** for both body and headings (`--font-heading: var(--font-sans)` in `index.css`). Two weights total.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Body | 16px (`text-base`) | 400 regular | 1.5 | All labels, help text, hex input value, card body copy, dialog body |
| Label | 14px (`text-sm`) | 500 medium | 1.5 | Form field labels above each input, contrast-badge text, action-bar "Unsaved changes" indicator |
| Heading | 20px (`text-xl`) | 600 semibold | 1.2 | `CardTitle` ("Identity", "Colors") |
| Display | 28px (`text-3xl`) | 600 semibold | 1.2 | Page title at top of `/settings` ("Settings") |

**Weight budget:** 2 weights total — **400 regular** + **600 semibold**. Label rows use the 500 variant of the same Geist file (Geist Variable supports continuous weight; 500 is the label sub-token, not an additional font face) — counted as part of the semibold track for the 2-weight rule.

**Decision:** Use `font-normal` (400) everywhere except headings/labels. Never apply `font-bold` (700) — it is out of contract.

## Color

This phase is a branding editor — the **user defines the final palette at runtime** via the 6 pickers. But the Settings page chrome itself must be painted from the existing shadcn neutral tokens so the editor is legible under every customer palette.

**60 / 30 / 10 split for the Settings page chrome (not the palette being edited):**

| Role | Token | Value (default, editable) | Usage |
|------|-------|---------------------------|-------|
| Dominant (60%) | `--background` / `bg-background` | `oklch(1 0 0)` (white, light mode) | Page background, scroll area, empty space around cards |
| Secondary (30%) | `--card` / `bg-card` | `oklch(1 0 0)` (light) with `--border` `oklch(0.922 0 0)` | Identity card, Colors card, sticky action bar surface |
| Accent (10%) | `--primary` / `bg-primary` | User-defined (default `oklch(0.205 0 0)`) | **Reserved for:** (1) the `Save` button, (2) the active focus ring on keyboard-focused inputs, (3) the `react-colorful` current-color dot indicator |
| Destructive | `--destructive` | User-defined (default `oklch(0.577 0.245 27.325)`) | **Reserved for:** (1) contrast-badge failure state, (2) `Reset to defaults` confirm dialog's confirm button, (3) `react-dropzone` rejection error toast border (via sonner default), (4) the unsaved-guard "Discard & leave" dialog primary action |

**Accent reserved for (explicit list — no other use allowed):**
1. The `Save` button (solid fill, `variant="default"`) in the sticky action bar
2. Focus rings on keyboard-focused inputs / pickers / dropzone (`focus-visible:ring-primary`)
3. The current-color dot inside each `react-colorful` picker (library default; do not override)

**Destructive reserved for:**
1. Contrast badge background when `wcagContrast < 4.5` (BRAND-08)
2. Confirm button in the **Reset to defaults** dialog (`variant="destructive"`)
3. Confirm button in the **Unsaved-changes guard** dialog's "Discard & leave" action (`variant="destructive"`)
4. Any `toast.error(...)` (sonner applies destructive styling automatically)

**Non-chrome rule:** Color pickers themselves render arbitrary user hex values — these are CONTENT, not chrome, and are exempt from the 60/30/10 split. A picker showing a bright purple swatch is correct output, not a violation.

**Dark mode:** The existing `.dark` variant in `index.css` stays untouched in Phase 6. Dark mode toggle is deferred to v1.2+.

## Interaction Contract

### Live preview (BRAND-07 / D-04 / D-05)

- Any draft change (color hex, app name) MUST call `queryClient.setQueryData(['settings'], nextDraft)` synchronously.
- No debouncing. CSS `setProperty` on 6 tokens is sub-millisecond.
- ThemeProvider's existing `useEffect` rewrites `:root` CSS vars + `document.title` — no new writer code in Phase 6.
- Logo upload bypasses the draft entirely: POSTs immediately, then refreshes the settings query so NavBar updates live.

### Dirty detection (D-07)

- `useSettingsDraft` compares draft vs snapshot across 8 fields: `color_primary`, `color_accent`, `color_background`, `color_foreground`, `color_muted`, `color_destructive`, `app_name`, `default_language`.
- `logo_url` and `logo_updated_at` are excluded (logo uploads immediately, has its own lifecycle).
- `isDirty` drives: Discard button visibility, Save button disabled state, "Unsaved changes" indicator visibility, unsaved-guard listener installation.

### Save flow (UX-02 / D-10)

1. User clicks `Save changes`
2. Container converts the 6 hex picker values to oklch via culori
3. `PUT /api/settings` with the full 8-field payload
4. **On 2xx:** `toast.success("Settings saved")` → update snapshot to response body → `queryClient.setQueryData(['settings'], response)` → Save button returns to disabled (pristine)
5. **On non-2xx:** `toast.error(...)` → draft PRESERVED → Save button stays enabled

### Reset flow (SET-04 surfaced here / D-12)

1. User clicks `Reset to defaults`
2. `ResetDialog` opens (NOT routed through unsaved-guard — see D-20)
3. **Cancel:** close dialog, no state change
4. **Confirm:** `PUT /api/settings` with `DEFAULT_SETTINGS` payload → on 2xx: snapshot ← response, cache ← response, `toast.success("Reset to default branding")` → NavBar immediately reflects default colors + text brand (logo cleared per Phase 4 D-07)

### Unsaved-changes guard (UX-01 / D-18, D-19, D-20)

- Installs listeners only while `isDirty === true`
- In-app nav intercept: document-level capture-phase click listener on `<a>` / `<Link>` elements; shows `UnsavedChangesDialog`
- Tab close: `window.addEventListener('beforeunload', e => { e.preventDefault() })`
- Dialog actions: `Stay` (cancel, remain on `/settings`) or `Discard & leave` (revert draft via `setQueryData(['settings'], snapshot)` THEN navigate)
- Intra-page interactions (opening Reset confirm, clicking a picker, editing a field) NEVER trigger the guard
- Cleanup: listeners removed on unmount and when `isDirty` returns to false

### Contrast badges (BRAND-08 / D-21, D-22, D-23)

- Compute on every render via `culori.wcagContrast(draftColor, pairedColor)`
- Three pairs:
  - `primary` vs `--primary-foreground` (read via `getComputedStyle(document.documentElement).getPropertyValue('--primary-foreground')`) → badge under the Primary picker
  - `background` vs `foreground` (both draft values) → badge under the Background picker AND under the Foreground picker
  - `destructive` vs literal `oklch(1 0 0)` (white) → badge under the Destructive picker
- Render badge only when `contrast < 4.5`
- Badge NEVER disables Save — warn-only per BRAND-08

### Focus management

- Every interactive element gets a visible `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2` (inherits from shadcn button defaults)
- `Dialog` traps focus (base-ui default) — do not override
- `Popover` inside `ColorPicker` traps focus while open (base-ui default)
