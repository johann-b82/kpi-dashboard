    - `buildMonthSpine("2024-10-01", "2025-03-01")` returns `["2024-10-01", "2024-11-01", "2024-12-01", "2025-01-01", "2025-02-01", "2025-03-01"]`
    - `buildMonthSpine("2025-06-01", "2025-06-01")` returns `["2025-06-01"]` (single month)
    - `mergeIntoSpine(["2025-01-01", "2025-02-01", "2025-03-01"], [{date: "2025-01-01", revenue: 100}, {date: "2025-03-01", revenue: 300}])` returns `[{date: "2025-01-01", revenue: 100}, {date: "2025-02-01", revenue: null}, {date: "2025-03-01", revenue: 300}]`
    - `formatMonthYear("2025-11-01", "en-US")` returns `"Nov '25"`
    - `formatMonthYear("2025-01-01", "de-DE")` returns `"Jan '25"`
    - `yearBoundaryDates(["2024-11-01", "2024-12-01", "2025-01-01", "2025-02-01"])` returns `["2025-01-01"]`
    - `yearBoundaryDates(["2025-03-01", "2025-04-01"])` returns `[]` (no January in spine)
  </behavior>
  <implementation>
    Create `frontend/src/lib/chartTimeUtils.ts` with these 4 exported functions:

    1. `buildMonthSpine(startDate: string, endDate: string): string[]` — iterate from start to end month, producing `YYYY-MM-01` strings. Parse year/month from input strings via `split("-")`.

    2. `mergeIntoSpine(spine: string[], points: ChartPoint[]): ChartPoint[]` — build a `Map` from `points` keyed by `date.slice(0,7)`, then map over spine returning `{ date, revenue: map.get(date.slice(0,7)) ?? null }`.

    3. `formatMonthYear(dateStr: string, locale: string): string` — create `new Date(dateStr)`, format month via `new Intl.DateTimeFormat(locale, { month: "short" })`, slice 2-digit year from `getFullYear()`, return `` `${month} '${year}` ``.

    4. `yearBoundaryDates(spine: string[]): string[]` — filter spine for entries where `d.slice(5, 7) === "01"`.

    Import `ChartPoint` from `@/lib/api`.

    For the test file: First check if vitest is in devDependencies (`cat frontend/package.json | grep vitest`). If not, run `cd frontend && npm install -D vitest` and ensure `package.json` has a `"test": "vitest"` script. Then write tests covering all behaviors listed above.
