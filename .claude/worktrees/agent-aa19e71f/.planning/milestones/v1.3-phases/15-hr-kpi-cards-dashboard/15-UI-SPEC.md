## Layout Contract

### Grid
- 5 KPI cards in a **3 + 2 grid**: `grid grid-cols-1 lg:grid-cols-3 gap-8`
- Top row (3 cards): Overtime Ratio, Sick Leave Ratio, Fluctuation
- Bottom row (2 cards, left-aligned): Skill Development, Revenue per Production Employee
- Bottom row uses same `grid-cols-1 lg:grid-cols-3` with 2 populated cells — matches Sales grid pattern

Source: CONTEXT.md D-04, D-05

### Banner placement
- No-sync banner: above the card grid, below the sync toolbar — `mb-6` separation from grid
- Error banner: above the card grid (same slot) — `mb-6`
- "nicht konfiguriert" state: within the affected card only — does not displace the grid

### Component hierarchy
```
HRPage.tsx
  └── sync toolbar (existing — unchanged)
  └── [banner: no-data or error — conditional]
  └── HrKpiCardGrid (new component)
       └── KpiCard × 5 (reuse existing)
            └── DeltaBadgeStack (reuse existing)
                 └── DeltaBadge × 2 (reuse existing)
```

Source: CONTEXT.md D-06, code_context section

## States & Interactions

### State matrix per card

| Condition | Card value | Delta badges | Extra element |
|-----------|-----------|--------------|---------------|
| Loading | skeleton pulse | skeleton pulse | — |
| Data present, deltas available | formatted value | `▲/▼ ±X%` in color | — |
| Data present, no prior period | formatted value | `—` (em-dash, muted, tooltip) | — |
| No sync yet (global) | `—` (em-dash) | `—` (em-dash) | Banner above grid |
| Setting not configured (card-specific) | `—` (em-dash) | hidden | "nicht konfiguriert" + link |
| Endpoint error (global) | `—` (em-dash) | `—` (em-dash) | Error banner above grid |

### Banner: no-sync state (D-07)
- Trigger: all 5 cards have `null` values (no sync ever run)
- Visual: muted info callout — `rounded-md border border-border bg-muted/40 p-4 mb-6`
- Copy: see Copywriting Contract below
- Cards remain visible with `—` values and no delta badges

### Banner: error state (D-09)
- Trigger: HR KPI endpoint returns HTTP error
- Visual: EXACT reuse of KpiCardGrid.tsx error block — `rounded-md border border-destructive bg-destructive/10 p-6`
- Copy: see Copywriting Contract below
- Cards remain visible with `—` values

### "nicht konfiguriert" card state (D-08)
- Trigger: card's required setting key is null/empty (`sick_leave_type_id`, `skill_attr_key`, or `production_dept`)
- Visual: card renders normally (KpiCard shell), value replaced by `—`, delta slot hidden, small muted text + Settings link below value
- Link: `<Link href="/settings">` — matches HRPage.tsx pattern using wouter

### Delta label format for fixed calendar month windows
- "vs. Vormonat" label: `vs. Vormonat` (DE) / `vs. prev. month` (EN) — static strings, no dynamic date (no preset bar)
- "vs. Vorjahr" label: `vs. Vorjahr` (DE) / `vs. prev. year` (EN) — static strings
- Rationale: HR tab has no time filter (CONTEXT.md D-03); labels are fixed, not derived from a date range picker

### Value formatting by KPI

| KPI | Format | Example |
|-----|--------|---------|
| Overtime Ratio | Percentage, 1 decimal | `12.4%` / `12,4 %` |
| Sick Leave Ratio | Percentage, 1 decimal | `3.2%` / `3,2 %` |
| Fluctuation | Percentage, 1 decimal | `8.7%` / `8,7 %` |
| Skill Development | Percentage, 1 decimal | `25.0%` / `25,0 %` |
| Revenue / Production Employee | Currency EUR, 0 decimals | `€12.450` / `12.450 €` |

Use `Intl.NumberFormat` — matches existing `formatCurrency` / `formatCount` pattern in KpiCardGrid.tsx.
