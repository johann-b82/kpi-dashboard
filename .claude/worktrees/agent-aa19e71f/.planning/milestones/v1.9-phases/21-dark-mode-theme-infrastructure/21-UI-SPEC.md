# Phase 21 — UI Design Contract

> Visual and interaction contract for the Dark Mode Theme Infrastructure phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase nature:** Infrastructure retheme. No new UI surfaces, no new components, no new copy. Contract locks the token-level behavior every existing surface must follow when the `.dark` class is toggled on `<html>`.

Scope reference: `.planning/phases/21-dark-mode-theme-infrastructure/21-CONTEXT.md` (decisions D-01..D-14)

## Color

Dark mode is a full surface inversion. The 60/30/10 split stays proportional; only the concrete hex values swap between the `:root` (light) and `.dark` CSS blocks in `frontend/src/index.css`.

### Light mode (`:root`, lines 60–93)

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | Page background, body |
| Secondary (30%) | `--card` / `--sidebar` / `--muted` | `oklch(1 0 0)` / `oklch(0.985 0 0)` / `oklch(0.97 0 0)` | Cards, navbar, sidebar, muted rows |
| Accent (10%) | `--primary` (brand, from Settings) | Brand value (default `oklch(0.55 0.15 250)`) | See "Accent reserved for" below |
| Destructive | `--destructive` | Brand value in light mode (default `oklch(0.577 0.245 27.325)`) | Destructive actions only |

### Dark mode (`.dark`, lines 95–127) — locked by D-05, D-06

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--background` | `oklch(0.145 0 0)` | Page background, body |
| Secondary (30%) | `--card` / `--sidebar` / `--muted` | `oklch(0.205 0 0)` / `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | Cards, navbar, sidebar, muted rows |
| Accent (10%) | `--primary` (brand, from Settings) | **Same brand value as light mode** (D-02, D-04, DM-04) | See "Accent reserved for" below |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` (shadcn dark default — D-04) | Destructive actions only |

### Accent reserved for (both modes — identical application)

Brand `--primary` / `--accent` is used for these elements only. Do NOT broaden during this retheme:

1. Primary buttons (submit, save, confirm)
2. Active state of `SegmentedControl` (DE/EN toggle, future Light/Dark toggle)
3. Active/selected navigation link in navbar
4. Focus ring on interactive elements (`--ring` inherits from primary via shadcn default, EXCEPT in dark mode where shadcn sets `--ring: oklch(0.556 0 0)` neutral; this phase keeps shadcn's neutral dark ring — acceptable per D-05)
5. Current-period line/bar in Recharts (`--color-chart-current` → `var(--primary)`)
6. KPI card delta-positive accent (already existing usage)
7. Chart axis emphasis when brand-highlighting is explicitly called for

**Not accent:** body text, card chrome, borders, hover states of secondary elements, chart gridlines, tooltip chrome.

### Semantic colors (identical in both modes — D-09)

| Role | Token | Value | Reason |
|------|-------|-------|--------|
| Warning / prior-period overlay | `--color-warning` | `#facc15` (amber) | Semantic consistency across modes (D-09). Do NOT override in `.dark`. |
| Success | `--color-success` | `#16a34a` | Semantic consistency; no dark override. |
| Chart current | `--color-chart-current` → `var(--primary)` | Brand (from Settings, identical both modes) | DM-04 |
| Chart prior | `--color-chart-prior` → `var(--muted)` | Mode-adaptive via token | Adapts automatically when `.dark` toggles |

## Recharts Contract (DM-03)

Every Recharts component must reference CSS tokens — zero hardcoded colors. This is the single largest deliverable of the phase (D-07, D-08).

| Chart element | Required token | Rendered via |
|---------------|----------------|--------------|
| `<CartesianGrid>` stroke | `var(--border)` | `stroke="var(--border)"` |
| `<XAxis>` / `<YAxis>` tick `fill` | `var(--muted-foreground)` | `tick={{ fill: 'var(--muted-foreground)' }}` |
| `<XAxis>` / `<YAxis>` axis `stroke` | `var(--border)` | `axisLine={{ stroke: 'var(--border)' }}` and `tickLine={{ stroke: 'var(--border)' }}` |
| `<Tooltip>` container | `var(--popover)` bg, `var(--popover-foreground)` text, `1px solid var(--border)` | `contentStyle={{ background: 'var(--popover)', color: 'var(--popover-foreground)', border: '1px solid var(--border)', borderRadius: 'var(--radius-md)' }}` |
| `<Tooltip>` label | `var(--popover-foreground)` | `labelStyle={{ color: 'var(--popover-foreground)' }}` |
| `<Tooltip>` item | `var(--popover-foreground)` | `itemStyle={{ color: 'var(--popover-foreground)' }}` |
| `<Tooltip>` cursor (hover bar highlight) | `var(--accent)` at reduced opacity | `cursor={{ fill: 'var(--accent)', opacity: 0.3 }}` |
| `<Legend>` text | `var(--muted-foreground)` | `wrapperStyle={{ color: 'var(--muted-foreground)' }}` |
| Current-period bar / line | `var(--color-chart-current)` (= `var(--primary)`) | `fill` / `stroke` |
| Prior-period overlay line/bar | `var(--color-warning)` (amber, identical both modes — D-09) | `stroke="var(--color-warning)"` |
| Reference line | `var(--border)` | `stroke="var(--border)"` |

**Audit rule (D-10):** Any `fill=`, `stroke=`, or inline style in `frontend/src/components/dashboard/**` or `frontend/src/components/hr/**` that contains a hex literal (`#...`), named color, `rgb()`, `hsl()`, or `oklch()` literal must be converted to one of the token references above. Exception: literals that already live inside `index.css`.

**Shared helper (Claude's discretion):** Extracting these defaults into a `frontend/src/lib/chartDefaults.ts` (exporting `axisProps`, `gridProps`, `tooltipProps`, `legendProps`) is permitted and encouraged. Either inline or shared is acceptable.

## Hardcoded Color Elimination (D-10, D-11)

This is a global audit deliverable. Every hardcoded color in `frontend/src/**` outside `index.css` must be replaced.

### Substitution table (non-negotiable)

| Hardcoded | Replace with | Dark-mode behavior |
|-----------|--------------|--------------------|
| `bg-white` | `bg-background` or `bg-card` (pick by role) | Auto-adapts |
| `bg-black` | `bg-foreground` | Auto-adapts |
| `text-black` / `text-gray-900` | `text-foreground` | Auto-adapts |
| `text-white` | `text-background` or `text-primary-foreground` (pick by role) | Auto-adapts |
| `text-gray-500` / `text-gray-600` | `text-muted-foreground` | Auto-adapts |
| `bg-gray-50` / `bg-gray-100` | `bg-muted` | Auto-adapts |
| `bg-gray-200` | `bg-secondary` or `bg-muted` (pick by role) | Auto-adapts |
| `border-gray-200` / `border-gray-300` | `border-border` | Auto-adapts |
| `border-black` | `border-foreground` | Auto-adapts |
| Hex literals (`#ffffff`, `#000`, `#f3f4f6`, etc.) | Token via `var(--token)` | Auto-adapts |
| Named CSS colors in inline styles (`white`, `black`, `red`) | Token via `var(--token)` | Auto-adapts |

### Grep patterns for the audit (Claude's discretion)

Reference patterns the planner/executor can use:

```
# Tailwind hardcoded utilities
rg -n "\b(bg|text|border|ring|fill|stroke)-(white|black|gray-\d+|slate-\d+|neutral-\d+|zinc-\d+)\b" frontend/src

# Hex literals in source (exclude index.css)
rg -n "#[0-9a-fA-F]{3,8}\b" frontend/src --glob '!**/index.css'

# Inline CSS color functions
rg -n "\b(rgb|rgba|hsl|hsla|oklch)\s*\(" frontend/src --glob '!**/index.css'

# Named CSS colors in inline styles
rg -n "(style=|: )\s*['\"](white|black|red|green|blue|yellow|gray|amber)['\"]" frontend/src
```

### Explicit exceptions

- `frontend/src/index.css` — the token source of truth; hex/oklch literals ARE the palette.
- `frontend/src/lib/defaults.ts` `DEFAULT_SETTINGS` — light-mode Settings default values (brand palette seed); these feed `THEME_TOKEN_MAP` and are applied only to light-mode surfaces per D-01.
- Settings color-picker preview swatches — show the literal brand color; these are NOT themed (confirmed out-of-scope in REQUIREMENTS.md).
- `#facc15` amber inside `index.css` `--color-warning` — intentional (D-09).

## Interaction Contract (Dark-Mode Specifics)

### ThemeProvider mode-awareness (D-01, D-03, D-14)

- ThemeProvider MUST detect current mode by reading `document.documentElement.classList.contains('dark')`.
- In **light mode**: apply all 6 Settings-driven tokens from `THEME_TOKEN_MAP` via `style.setProperty` on `<html>`.
- In **dark mode**: apply ONLY `--primary` and `--accent` inline (D-02, DM-04). Do NOT inline-set `--background`, `--foreground`, `--muted`, `--destructive` — the `.dark` CSS block must win.
- When `.dark` is added/removed externally (devtools in Phase 21, toggle in Phase 22), ThemeProvider must re-evaluate and remove inline surface tokens on the dark transition, re-apply them on the light transition.
- Mechanism (executor's choice): `MutationObserver` on `<html>`'s `class` attribute, or subscription to the Phase 22 preference state. Either is acceptable.

### Focus ring

- Light mode: `--ring` = `oklch(0.708 0 0)` (shadcn default, neutral)
- Dark mode: `--ring` = `oklch(0.556 0 0)` (shadcn default, neutral-mid)
- Focus outline must remain visible on cards, inputs, buttons in both modes. No changes to focus-ring utility classes in this phase.

### Shadows and elevation

Shadcn defaults (`shadow-sm`, `shadow-md`) are rgba-black with alpha — they darken both modes visually without token work. Do NOT add a `dark:` shadow override. Acceptable trade-off per D-05 (shadcn defaults as-is).

### Border visibility

Dark-mode borders use `oklch(1 0 0 / 10%)` — intentionally subtle. Cards rely on the card/background surface contrast (not borders) for separation. Executor must NOT bump border opacity to make cards "pop" in dark mode; this would break D-05.

### Brand accent verification (DM-04)

Acceptance test: toggle `.dark` class via devtools on a page containing a primary button and a current-period chart line. Computed color of `background-color` on the button and `stroke` on the line MUST be identical strings before and after the toggle. If they differ, ThemeProvider is leaking or the chart hardcoded a color.
