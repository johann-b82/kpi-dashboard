  <action>
    Create `frontend/src/bootstrap.ts` that exports `async function bootstrap(): Promise<void>`. It must:

    1. Be **idempotent** — guard against double-init via a module-level `let bootstrapPromise: Promise<void> | null = null;` (Research §Pattern 1 sketch).
    2. First `await i18nInitPromise` (defensive — Research Open Q3).
    3. Then `await fetchSettings()` — reuse the existing function, do NOT duplicate fetch logic (D-05).
    4. On success: call `await i18n.changeLanguage(settings.default_language.toLowerCase())` AND `queryClient.setQueryData(["settings"], settings)` (D-01, D-06).
    5. On error: catch the error, log a `console.warn` with `[bootstrap]` prefix, call `await i18n.changeLanguage("en")` as the hardcoded fallback (D-04), and return normally. Do NOT rethrow. Do NOT seed the cache with `DEFAULT_SETTINGS` — let the App-level `useSettings()` do a real fetch so the error surfaces via the existing toast/error-boundary path (Research §Pattern 1 comment).
    6. Do NOT use `react-i18next` hooks — bootstrap runs before React mounts.

    Reference shape (copy-paste, adapt imports to actual file paths):

    ```typescript
    import i18n, { i18nInitPromise } from "./i18n";
    import { queryClient } from "./queryClient";
    import { fetchSettings } from "./lib/api";

    // Guard against double-init (hot reload, StrictMode effects, etc.).
    let bootstrapPromise: Promise<void> | null = null;

    export function bootstrap(): Promise<void> {
      if (bootstrapPromise) return bootstrapPromise;
      bootstrapPromise = (async () => {
        await i18nInitPromise;
        try {
          const settings = await fetchSettings();
          await i18n.changeLanguage(settings.default_language.toLowerCase());
          queryClient.setQueryData(["settings"], settings);
        } catch (err) {
          // D-04: fallback — do NOT rethrow. App renders, useSettings() surfaces
          // the real error via the normal query error path. Intentionally do NOT
          // seed the cache with DEFAULT_SETTINGS — that would mask the error.
          // eslint-disable-next-line no-console
          console.warn("[bootstrap] fetchSettings failed, falling back to EN:", err);
          await i18n.changeLanguage("en");
        }
      })();
      return bootstrapPromise;
    }
    ```

    If `queryClient` lives at a different path (e.g. `./lib/queryClient`), adjust the import; the Task 1 read_first list includes the file to locate it. Use `Settings` type from `lib/api.ts` implicitly (inferred from `fetchSettings` return).
  </action>
