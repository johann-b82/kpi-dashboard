# Phase 35 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Character

Phase 35 is a **pure content-authoring phase**. No new components, routes, or CSS are introduced. All rendering infrastructure (MarkdownRenderer, Tailwind Typography prose styles, syntax highlighting, TOC extraction, sidebar wiring) was delivered in Phases 33–34 and is consumed as-is.

The design contract for this phase is therefore a **Markdown authoring contract** — it governs the document structure, prose style, callout syntax, and cross-reference pattern that the executor must follow when writing the 10 article files (5 articles × 2 languages).

## Typography

All article typography is rendered by Tailwind Typography `prose` — no new font declarations needed. The MarkdownRenderer applies `prose dark:prose-invert max-w-none` and passes Markdown headings through `HeadingWithAnchor` for anchor links.

| Role | Markdown Element | Rendered By |
|------|-----------------|-------------|
| Article title | `# Title` (one per article) | `prose h1` — Tailwind Typography |
| Section heading (TOC-generating) | `## Section` | `HeadingWithAnchor h2` — includes anchor icon |
| Sub-section (sparingly) | `### Sub` | `HeadingWithAnchor h3` — includes anchor icon |
| Body prose | Paragraphs | `prose p` — 16px, line-height 1.75 (Tailwind Typography default) |
| Inline code / UI labels | `` `label` `` | `prose code` — monospace, subtle background |
| Tip / Note callout | `> **Tip:**` blockquote | `prose blockquote` — left border, muted text |

Heading convention (locked — from D-05 + Phase 33 `extractToc` requirement):
- `#` — article title only (one per file)
- `##` — all main sections (generates TOC entries)
- `###` — sub-sections only when nesting is genuinely required; use sparingly

No new color tokens are introduced. All colors are provided by the existing shadcn `base-nova / neutral` theme via CSS variables. Dark mode is handled by `prose-invert` already applied in `MarkdownRenderer.tsx`.

| Role | Value | Usage |

Accent reserved for: heading anchor link icons (hover reveal), active sidebar navigation item. Not used for body text, callouts, or standard links.

## Copywriting Contract

This is the primary design contract for Phase 35. All article content must conform to these rules.

### Article Structure Template (D-05 — locked)

Every article must follow this exact structure:

```
# {Article Title}

{One paragraph introduction — states what the article covers and when to use it.}

## {Section 1 Heading}

{Step-by-step numbered list or explanatory prose.}

## {Section 2 Heading}

> **Tip:** {Actionable tip relevant to this section.}

## Related Articles

- [Article Name](/docs/user-guide/{slug})
```

### Tone Rules (D-01, D-02 — locked)

- **Tutorial-style:** numbered steps for all procedural instructions (e.g., "1. Click the Browse button.")
- **Professional casual:** friendly, direct, no jargon. Example correct: "Upload your file by dragging it onto the drop zone." Example incorrect: "Utilize the drag-and-drop interface to initiate the file upload process."
- **No screenshots:** text-only. Use bold (`**label**`) for UI element names, code backticks for exact strings.

### UI Label Rule (Pitfall 1 — from RESEARCH.md)

All button labels, field names, and status text in articles MUST match the exact strings in `frontend/src/locales/en.json` (EN articles) and `frontend/src/locales/de.json` (DE articles). Do not paraphrase UI labels.

### Callout Syntax (D-04 — locked)

| Callout Type | Markdown Syntax |
|-------------|----------------|
| Tip | `> **Tip:** {text}` |
| Note | `> **Note:** {text}` |
| Warning | `> **Warning:** {text}` |

### Cross-Reference Syntax (D-06 — locked)

Inline Markdown links using absolute paths:
```markdown
See [Filters & Date Ranges](/docs/user-guide/filters) for details.
```
