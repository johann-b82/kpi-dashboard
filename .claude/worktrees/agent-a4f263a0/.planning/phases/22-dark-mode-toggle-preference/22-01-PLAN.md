<task type="auto">
  <name>Task 1: Add inline pre-hydration theme script to index.html</name>
  <files>frontend/index.html</files>
  <action>
    Insert a synchronous inline `<script>` block inside `<head>` BEFORE the existing `<style>` tag and BEFORE `<script type="module" src="/src/main.tsx">`. The script must execute before any CSS paints and before React mounts to satisfy D-04 / D-05 (no FOUT).

    Exact script logic (per D-04, D-06, D-10, D-11):

    ```html
    <script>
      (function () {
        try {
          var stored = localStorage.getItem('theme');
          var isDark;
          if (stored === 'dark') {
            isDark = true;
          } else if (stored === 'light') {
            isDark = false;
          } else {
            // No explicit choice OR invalid value (D-10, D-11) — fall back to OS
            isDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
          }
          if (isDark) {
            document.documentElement.classList.add('dark');
          } else {
            document.documentElement.classList.remove('dark');
          }
        } catch (e) {
          // localStorage access can throw in privacy mode / sandboxed iframes — fail silent, default light
        }
      })();
    </script>
    ```

    Constraints:
    - MUST be placed in `<head>` BEFORE the `<style>` block so the .dark class is set before any CSS is applied.
    - MUST be inline (no `src=`) per D-05 — external scripts would race with paint.
    - MUST be a plain `<script>` (no `type="module"`, no `defer`, no `async`) — module/async scripts are deferred until after parsing, defeating the purpose.
    - MUST use `var` and IIFE (not `let`/`const`/arrow) — minor: defensive against any unexpected build pre-processing; consistent with "this is hand-written, not bundled."
    - MUST wrap in try/catch — localStorage throws in some sandboxed contexts (D-11 spirit).
    - Do NOT touch the existing `bootstrap-splash` styles. Note: the splash uses hardcoded `#ffffff` and `#64748b` — these are pre-existing and OUT OF SCOPE for Phase 22 (would belong in Phase 23 contrast audit if anywhere).

    Per D-13: Phase 21's ThemeProvider already MutationObserver-watches `<html>.class`, so as soon as this script sets `.dark`, ThemeProvider's effect (after React mounts) will pick it up and apply correct surface tokens.
  </action>
