## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline gaps |
| sm | 8px | Button inner padding vertical, compact table cell padding |
| md | 16px | Card padding, input inner padding, default element spacing |
| lg | 24px | Drop zone internal padding, section spacing within the upload page |
| xl | 32px | Gap between drop zone and upload history section |
| 2xl | 48px | Top/bottom page padding |
| 3xl | 64px | Max content width left/right margin (not used as gap) |

Exceptions:
- Drop zone minimum height: 160px (functional minimum for drag target — not a spacing token)
- Touch/click targets: minimum 44px height for all interactive controls (Button, table row delete button)

**Source:** 8-point grid default; D-14 (centered drop zone layout).

## Typography

| Role | Size | Weight | Line Height | Treatment |
|------|------|--------|-------------|-----------|
| Body | 14px | 400 (regular) | 1.5 | — |
| Label | 14px | 400 (regular) | 1.4 | uppercase, letter-spacing 0.05em |
| Heading | 20px | 600 (semibold) | 1.2 | — |

**Active weight set: 400 (regular) and 600 (semibold). No other weights used in Phase 2.**

**Applied mapping:**
- Body (14px/400): Table cell content, error list items, toast body text
- Label (14px/400/uppercase+tracking): Table column headers, form field labels, badge text, language toggle label. Uppercase and letter-spacing distinguish Label from Body without a separate weight.
- Heading (20px/600): Page title ("Datei hochladen" / "Upload File"), upload history section title

**Display (28px) is not used in Phase 2 — reserved for Phase 3 dashboard metric cards. No weight declaration is made for it in this phase.**

**Source:** Tailwind CSS v4 defaults; internal tool context (compact information density preferred). Revised: 4-weight set reduced to 2-weight set per checker constraint.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — white | Page background, card backgrounds, table row background |
| Secondary (30%) | `hsl(210 40% 96.1%)` — slate-100 | Drop zone background (idle state), table header row, sidebar/nav if added in Phase 3 |
| Accent (10%) | `hsl(221.2 83.2% 53.3%)` — blue-600 | Upload button, active drag-over drop zone border, spinner |
| Destructive | `hsl(0 72.2% 50.6%)` — red-600 | Delete button, 'failed' status badge, parse error list border |

**Accent reserved for:** Upload CTA button (primary action), drop zone active border when `isDragActive === true`, loading spinner during upload. Never use accent for status indicators, nav links, or secondary actions.

**Additional semantic colors:**
- Success state: `hsl(142.1 76.2% 36.3%)` — green-600. Used for: 'success' status badge only.
- Warning state: `hsl(47.9 95.8% 53.1%)` — yellow-400. Used for: 'partial' status badge only.
- Muted text: `hsl(215.4 16.3% 46.9%)` — slate-500. Used for: empty state body, placeholder text, table secondary values (timestamps, error counts when zero).

**Source:** shadcn default slate CSS variable set; CONTEXT.md D-11 (three upload statuses: success/partial/failed).

## Interaction States

### Drop Zone

| State | Visual |
|-------|--------|
| Idle | slate-100 background, slate-300 dashed border (2px), muted text |
| Drag active (`isDragActive`) | blue-50 background, blue-600 solid border (2px), body text changes to blue-600 |
| Uploading/parsing | Drop zone disabled, Loader2 spinner (blue-600, 24px, animate-spin) centered, text "Wird verarbeitet..." / "Processing..." |
| Success | Returns to idle state; toast fires (see Copywriting) |
| Error | Returns to idle state; error list appears below drop zone |
| Wrong file type dropped | Red-600 dashed border (2px) for 500ms, then returns to idle; inline message below drop zone |

### Upload History Table

| State | Visual |
|-------|--------|
| Loading | Skeleton rows (3 rows, full width, slate-200 animated pulse) |
| Empty | Empty state (see Copywriting) |
| Populated | Standard table; status badges colored per semantic colors |
| Delete hover | Delete button (ghost variant, red-600 on hover) |
| Delete confirm open | Dialog modal (shadcn Dialog), backdrop overlay |

### Status Badges

| Status | Badge color |
|--------|-------------|
| success | green-600 background, white text |
| partial | yellow-400 background, slate-900 text |
| failed | red-600 background, white text |

## Copywriting Contract

All copy is bilingual. Format: German / English (language toggle controls which is shown at runtime via react-i18next).

| Element | German | English |
|---------|--------|---------|
| Page title (h1) | Datei hochladen | Upload File |
| Drop zone prompt | Datei hier ablegen | Drop file here |
| Drop zone sub-prompt | oder | or |
| Browse button | Datei auswählen | Browse |
| Accepted formats hint | Akzeptiert: .csv, .txt | Accepts: .csv, .txt |
| Processing state | Wird verarbeitet... | Processing... |
| Primary CTA (success toast heading) | Datei hochgeladen | File uploaded |
| Success toast body | {filename}: {n} Zeilen importiert | {filename}: {n} rows imported |
| Partial toast body | {filename}: {n} Zeilen importiert, {m} Zeilen übersprungen | {filename}: {n} rows imported, {m} rows skipped |
| Invalid file type error | Nicht unterstütztes Format: {ext}. Nur .csv und .txt erlaubt. | Unsupported format: {ext}. Only .csv and .txt allowed. |
| Validation error heading | Importfehler ({n} Zeilen übersprungen) | Import errors ({n} rows skipped) |
| Validation error row format | Zeile {row}, {column}: {message} | Row {row}, {column}: {message} |
| Upload history section title | Upload-Verlauf | Upload History |
| Table column: filename | Dateiname | Filename |
| Table column: timestamp | Hochgeladen am | Uploaded at |
| Table column: row count | Zeilen | Rows |
| Table column: status | Status | Status |
| Table column: error count | Fehler | Errors |
| Table column: delete action | (icon only — Trash2 icon, aria-label bilingual) | — |
| Empty state heading | Noch keine Uploads | No uploads yet |
| Empty state body | Lade eine .csv oder .txt Datei hoch, um zu beginnen. | Upload a .csv or .txt file to get started. |
| Delete confirmation dialog title | Upload löschen | Delete upload |
| Delete confirmation body | "{filename}" und {n} zugehörige Datensätze werden dauerhaft gelöscht. | "{filename}" and {n} associated records will be permanently deleted. |
| Delete confirmation CTA | Löschen | Delete |
| Delete cancel button | Upload behalten | Keep upload |
| Language toggle label | DE / EN | DE / EN |

**Copywriting note:** The delete cancel button uses "Upload behalten" (DE) / "Keep upload" (EN) — a noun-verb label that names the retained object and clarifies the outcome. Generic "Abbrechen" / "Cancel" is not used.

**Source:** CONTEXT.md D-13, D-14, D-17, D-18, D-19, D-20; REQUIREMENTS.md UPLD-02, UPLD-04, MGMT-01. Revised: delete cancel button label updated per checker Dimension 1 finding.

## i18n Contract

- Default language: German (`lng: "de"`)
- Fallback language: English (`fallbackLng: "en"`)
- Translation files: `frontend/src/locales/de.json` and `frontend/src/locales/en.json`
- Library: react-i18next 17.0.2 + i18next 26.0.4
- All UI strings use `t("key")` — no hardcoded German or English strings in components
- Data content (filenames, column names, ERP values) is NOT translated — displayed as-is

**Source:** CONTEXT.md D-18; RESEARCH.md Supporting Libraries.
