Per D-01, add a new endpoint `GET /api/settings/logo/public` that serves the uploaded logo without requiring authentication.

In `backend/app/routers/settings.py`, add a new standalone route OUTSIDE the existing `router` (which has `dependencies=[Depends(get_current_user)]`). Create a second router:

```python
public_router = APIRouter(prefix="/api/settings", tags=["settings"])
```

Add this endpoint on `public_router`:

```python
@public_router.get("/logo/public")
async def get_logo_public(
    request: Request,
    db: AsyncSession = Depends(get_async_db_session),
) -> Response:
    row = await _get_settings_row(db)
    if row.logo_data is None or row.logo_updated_at is None:
        raise HTTPException(status_code=404, detail="No logo set")
    etag = _etag(row)
    if request.headers.get("if-none-match") == etag:
        return Response(status_code=304, headers={"ETag": etag})
    return Response(
        content=row.logo_data,
        media_type=row.logo_mime or "application/octet-stream",
        headers={
            "ETag": etag,
            "Cache-Control": "public, max-age=300",
        },
    )
```

Then in `backend/app/main.py`, import and include the public_router:

```python
from app.routers.settings import router as settings_router, public_router as settings_public_router
```

Add `app.include_router(settings_public_router)` alongside the existing settings_router include.

The endpoint reuses the existing `_get_settings_row` and `_etag` helper functions. It returns the same logo bytes as the authenticated endpoint but with `Cache-Control: public, max-age=300` for browser caching. Returns 404 when no logo is uploaded (per D-02, login page handles this by showing text only).
