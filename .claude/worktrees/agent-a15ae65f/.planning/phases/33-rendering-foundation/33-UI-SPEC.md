## Typography

All sizes use Geist Variable. The `@tailwindcss/typography` plugin drives prose styles via `prose` / `dark:prose-invert` classes. Values below reflect the plugin's defaults aligned to this project's scale — do not override unless specified.

Exactly 4 declared sizes: 14px, 16px, 20px, 28px.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body prose (p, li) | 16px | 400 | 1.75 (prose default) | |
| Code inline | 14px | 400 | 1.5 | |
| Code block | 14px | 400 | 1.6 | Same size as code inline; monospace font differentiates visually |
| H1 article title | 28px | 600 | 1.2 | |
| H2 section heading | 20px | 600 | 1.3 | |
| H3 subsection | 16px | 600 | 1.4 | |
| TOC label (nav item) | 14px | 400 | 1.4 | |
| TOC section title | 14px | 600 | 1.2 | Uppercase + `tracking-wide` differentiates from nav items — no distinct size needed |

### Syntax Highlighting (highlight.js themes)

| Mode | Theme |
|------|-------|
| Light | `github` (light, minimal) |
| Dark | `github-dark` (matches dark token palette) |

Theme swap driven by the existing `.dark` class strategy on `<html>`. Import both themes; activate via CSS:

```css
/* light default */
@import 'highlight.js/styles/github.css';
/* dark override */
.dark @import 'highlight.js/styles/github-dark.css';
```

Or use a single CSS layer with `.dark pre code.hljs { ... }` overrides if Vite bundling requires it.

## Layout Contract

### Two-Column Docs Layout (desktop)

```
┌──────────────────────────────────────┬──────────────────┐
│  Article content (prose)             │  TOC sidebar     │
│  max-w: ~720px                       │  width: 240px    │
│  flex-1, grows                       │  sticky top-24   │
└──────────────────────────────────────┴──────────────────┘
```

- Breakpoint for TOC collapse: `lg` (1024px). Below `lg`, TOC is hidden (`hidden lg:block`).
- Article column: `flex-1 min-w-0` to prevent prose overflow in flex context.
- TOC sidebar: `sticky top-24 hidden lg:block w-60 shrink-0`
- Outer wrapper: `flex gap-8 px-6 py-8` within the lazy-loaded `DocsPage` container.

### Heading Anchors

- Anchor `id` generated from heading text via `remark-slug` (or `rehype-slug`) — lowercase, hyphen-separated.
- Anchor link rendered as a `#` symbol (lucide `Link` icon, 14px) appearing on heading `:hover`.
- Icon is visually hidden by default (`opacity-0 group-hover:opacity-100`) — not hidden from screen readers (use `aria-label="Link to section"`).
- Clicking the anchor updates `window.location.hash` and scrolls to the heading.

### Table of Contents

- Generated by scanning the parsed Markdown AST for `h2` and `h3` nodes (post-remark parse, pre-render).
- Renders as a `<nav aria-label="Table of contents">` with an `<ol>` list.
- H2 items: full left-edge, font-size 14px weight 400.
- H3 items: `pl-4` indent, font-size 14px weight 400.
- Active item (intersection observer on heading): left border `border-l-2 border-primary text-primary`.
- Inactive item: `text-muted-foreground hover:text-foreground`.
- TOC section label above list: "On this page" (EN) / "Auf dieser Seite" (DE) — i18n key `docs.toc.title`. Weight 600, 14px, uppercase, tracking-wide.

### Lazy Loading

- Route `/docs` added to `App.tsx` via `React.lazy()` + `Suspense`.
- Suspense fallback: centered `<Loader2 className="animate-spin" />` (lucide, 24px) with `aria-label="Loading documentation"`.
- No skeleton — docs chunk loads once and is cached; spinner duration is negligible.
