<objective>
Wire `https://kpi.internal` end-to-end: Vite accepts forwarded `Host: kpi.internal` requests, HMR reconnects over WSS through NPM, and host-side direct exposure of frontend:5173 is removed (NPM becomes the sole edge).

Purpose: kpi.internal is the flagship hostname for Phase 26 — the full KPI Light app must function through it, including dev HMR, because D-07/D-08 say dev and prod share one compose file with identical routing. Without HMR-over-HTTPS configured, editing `.tsx` files would fall back to full page reloads in dev.
Output: Updated `frontend/vite.config.ts` (HMR + allowedHosts), `docker-compose.yml` with `frontend` direct port bindings commented out, and a manually-created NPM proxy host for `kpi.internal` (captured in the checkpoint, persisted to the `npm_data` volume from 26-01).

<task type="auto">
  <name>Task 1: Update Vite config for HMR-over-HTTPS behind NPM and allowedHosts whitelist</name>
  <files>frontend/vite.config.ts</files>
  <action>
Modify `server` block in `frontend/vite.config.ts` so Vite works when reached via NPM at `https://kpi.internal`:

1. Add `allowedHosts: ["kpi.internal", "localhost"]` — per D-06. Including `localhost` keeps direct 5173 access working for the commented-out debug bind path (Claude's Discretion in 26-CONTEXT.md). Do NOT use `allowedHosts: true`; that disables the SSRF protection Vite added in 5.0.12+.
2. Add `hmr: { clientPort: 443, protocol: "wss", host: "kpi.internal" }` — per D-05. This tells the browser-side HMR client to open its WebSocket to `wss://kpi.internal:443` (which NPM terminates and proxies to frontend:5173 with websocket upgrade) instead of trying `ws://localhost:5173` directly.
3. Preserve the existing `server.host`, `server.proxy` settings exactly as-is — the `/api` dev proxy is still used in direct-debug mode and must not regress.

The final `server` block should read (approximately):
```ts
server: {
  host: "0.0.0.0",
  allowedHosts: ["kpi.internal", "localhost"],
  hmr: {
    clientPort: 443,
    protocol: "wss",
    host: "kpi.internal",
  },
  proxy: {
    "/api": {
      target: process.env.VITE_API_TARGET || "http://api:8000",
      changeOrigin: true,
    },
  },
},
```

Reference: D-05, D-06 in 26-CONTEXT.md and Vite HMR docs (https://vitejs.dev/config/server-options.html#server-hmr).
  </action>

<automated>cd frontend && npx --no-install tsc --noEmit -p tsconfig.json 2>&1 | tee /tmp/tsc.out; grep -E 'allowedHosts|clientPort|wss' vite.config.ts</automated>
  </verify>
  <done>`vite.config.ts` type-checks clean, `allowedHosts` includes both `kpi.internal` and `localhost`, `hmr` block has `clientPort: 443` + `protocol: "wss"` + `host: "kpi.internal"`, and the existing `/api` dev proxy is unchanged.</done>
