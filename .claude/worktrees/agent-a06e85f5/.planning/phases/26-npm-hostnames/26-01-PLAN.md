<task type="auto">
  <name>Task 2: Add npm service + frontend healthcheck to docker-compose.yml (healthcheck-gated edge)</name>
  <files>docker-compose.yml, .env.example</files>
  <action>
Extend `docker-compose.yml` with (a) a healthcheck on the existing `frontend` service and (b) a new `npm` service gated on both `api` and `frontend` being healthy. This satisfies INF-05 ("healthcheck-gated dependency order" for the full stack) and closes the brief 502 window that would otherwise exist while Vite compiles on first boot.

### Part A — Add a healthcheck to the existing `frontend` service

The `frontend` image is `node:22-alpine` (see frontend/Dockerfile). Alpine ships busybox `wget` but NOT `curl` by default — use `wget`. Add to the existing `frontend` service block:

```yaml
frontend:
  # ... existing fields (build, depends_on api healthy, ports, env, etc.) unchanged ...
  healthcheck:
    test: ["CMD-SHELL", "wget -q --spider http://localhost:5173 || exit 1"]
    interval: 5s
    timeout: 5s
    retries: 10
    start_period: 30s
```

Rationale:
- Vite dev server has no `/health` endpoint; a TCP/HTTP probe of `:5173` is the honest check — once Vite is listening on 5173 it will serve the index HTML, which is exactly what NPM will proxy.
- `start_period: 30s` accounts for `npm install` + Vite cold-start on first boot; after that the container restarts in a few seconds.
- `wget -q --spider` on busybox returns exit 0 on any 2xx/3xx, nonzero on connection failure or 4xx/5xx — correct semantics for readiness.

Do NOT remove `frontend`'s existing `ports: - "5173:5173"` here; that's handled in Plan 26-02.

### Part B — Add the `npm` service

Pin the image to a specific 2.x tag — do NOT use `:latest` (CLAUDE.md: pinned versions are house style). Recommended: `jc21/nginx-proxy-manager:2.11.3` (current stable). The service must:

1. Expose ports `80:80`, `443:443`, and `81:81` on the host. Port 81 is the admin UI (D-09). Ports 80/443 are the HTTPS edge for kpi.internal et al.
2. Mount two named volumes for persistence (D-11):
   - `npm_data:/data`
   - `npm_letsencrypt:/etc/letsencrypt`
3. Bind-mount `./certs:/etc/certs:ro` so NPM proxy-host entries can reference the mkcert-generated cert+key. Read-only (`:ro`) to prevent NPM writing into the host cert dir.
4. Declare a healthcheck that probes the admin UI. The NPM image has `curl` available, so: `test: ["CMD-SHELL", "curl -f http://localhost:81 || exit 1"]` with interval/timeout/retries matching the api service pattern (10s / 5s / 3, start_period 30s — NPM takes longer than the api to become healthy on first boot).
5. `depends_on` block (INF-05 — both upstreams must be healthy before NPM starts accepting traffic; otherwise NPM would front a 502 window while Vite compiles):
   ```yaml
   depends_on:
     api:
       condition: service_healthy
     frontend:
       condition: service_healthy
   ```
6. Add `restart: unless-stopped` so NPM survives docker daemon restarts on the single-VM prod host.

Register both new volumes in the top-level `volumes:` block alongside `postgres_data`:
```yaml
volumes:
  postgres_data:
  npm_data:
  npm_letsencrypt:
```

DO NOT change any other service in this task beyond adding the frontend healthcheck — `frontend` port bindings stay in place for now. Removing `5173:5173` is done in Plan 26-02 after Vite HMR is reconfigured. Touching it here would break local dev before the replacement is in place.
