## Goal

Deploy an Outline wiki instance alongside KPI Dashboard as reusable multi-project documentation infrastructure, secured by a lightweight Dex OIDC IdP that both apps trust, and seed the wiki with KPI Dashboard's developer + user docs.

## Key Accomplishments

1. **NPM edge TLS** with mkcert wildcard certs + three internal hostnames (kpi.internal, wiki.internal, auth.internal)
2. **Dex OIDC IdP** deployed at auth.internal with SQLite persistence, two seeded users, bcrypt rotation workflow via python-bcrypt (Dex v2.43 hash-password subcommand was removed)
3. **KPI Dashboard SSO integration** — SessionMiddleware, authlib Dex client (PKCE + offline_access), app_users upsert table, get_current_user dependency on 6 business routers, DISABLE_AUTH dev bypass, frontend useCurrentUser/ProtectedRoute/NavBar logout
4. **Outline wiki** at wiki.internal — dedicated outline-db (Postgres 17) + outline-redis (Redis 7) + local file storage, split-endpoint OIDC (browser auth.internal for auth-uri, internal dex:5556 for token/userinfo), mkcert CA mounted for issuer validation
5. **NavBar wiki icon** — BookOpen between LanguageToggle and Upload, with nav.wiki i18n and VITE_WIKI_URL env override
6. **Product rebrand** KPI Light → KPI Dashboard (Phase 30.1, display name only — internal identifiers preserved to avoid stop-the-world migration)
7. **9-page documentation collection** in Outline (API-seeded) mirrored to docs/wiki-seed/ with Mermaid architecture diagram, thin API reference page (links to live Swagger UI), WMP-03 multi-project onboarding recipe

## Known Limitations (Shipped as Documented)

- **No silent cross-app SSO** — Dex staticPasswords connector does not set a browser session cookie; each OIDC client re-prompts for the password even in the same browser. Value delivered = shared credentials, not one-click SSO. Upgrading to an upstream connector (LDAP, OIDC, SAML) is the standard fix — out of scope for v1.11.
- **No RP-initiated logout** — Dex v2.43 does not expose `end_session_endpoint` (upstream issue #1697); per-app logout works but can't invalidate Dex's own session. Mitigation: 1h ID-token TTL.

## Bugs Caught Mid-Milestone

1. **Dex `${VAR}` expansion** does NOT work in `staticClients.secret` at config load — fixed with `envsubst` preprocessing at dex container start (commit 6aaaa94). Unblocked both KPI Dashboard and Outline OIDC login.
2. **Product rename needed mid-milestone** — handled via inserted decimal Phase 30.1 before Phase 31 authoring began, so the seed docs landed with the new name from day one.
