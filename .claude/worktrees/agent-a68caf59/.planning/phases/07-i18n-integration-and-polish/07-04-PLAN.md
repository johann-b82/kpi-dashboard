<objective>
Upgrade the existing NavBar `LanguageToggle` component from an ephemeral language toggle (Phase 5) into a persisting, immediate-PUT toggle (D-12). On click, it reads current settings from the TanStack cache, builds the full 8-field PUT payload, fires `updateSettings()` via `useMutation`, writes the response back to cache, calls `i18n.changeLanguage`, and shows a success/error toast. When on `/settings` AND `useSettingsDraftStatus().isDirty === true`, the toggle is disabled with a tooltip (D-13). On non-Settings routes, the toggle is always enabled (D-14).

Purpose: Completes I18N-01 cross-cut — users can change the language from either the Settings page (plan 07-03) OR directly from the NavBar on any other page. Enforces the single-writer invariant: one writer at a time (toggle disabled while Settings draft is dirty).

Output: Rewritten `LanguageToggle.tsx` with TanStack useMutation + sonner toast + context-aware disable state.
</objective>
