## Spacing Scale

Declared values (multiples of 4 only). Pre-populated from existing layout patterns in `NavBar.tsx` and `DocsPage.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing — navbar px-6 = 24px via lg token |
| lg | 24px | Section padding — `px-6 py-8` on DocsPage, `px-6` on NavBar |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Navbar height: fixed `h-16` (64px) — matches existing `NavBar.tsx`
- Touch target for navbar icon links: `p-2` + `h-5 w-5` icon = ~36px effective hit area (consistent with existing UploadIcon, SettingsIcon pattern)
- Docs page top offset for sticky TOC: `top-24` (96px) — existing `DocsPage.tsx` pattern, accounts for fixed 64px navbar
- DocsSidebar width: `w-56` (224px) — derived from RESEARCH.md `DocsSidebar` pattern; matches TableOfContents `w-60` (240px) on the right
- Three-column gap: `gap-8` (32px) — existing `DocsPage.tsx` gap pattern

## Color

Pre-populated from `index.css` CSS custom properties. All values are OKLCH tokens resolved via Tailwind CSS v4's `@theme inline` block.

| Role | Token | Light value | Dark value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) white | oklch(0.145 0 0) near-black | Page background, main content area |
| Secondary (30%) | `--card` / `--sidebar` | oklch(1 0 0) / oklch(0.985 0 0) | oklch(0.205 0 0) / oklch(0.205 0 0) | Navbar (`bg-card`), DocsSidebar background |
| Accent (10%) | `--primary` | oklch(0.205 0 0) near-black | oklch(0.922 0 0) near-white | Reserved for active state only (see below) |

Accent (`text-primary`) reserved for:
- Active navbar icon (docs Library icon when on any `/docs/*` route)
- Active sidebar article link (currently selected article)

Accent NOT used for: hover states (use `hover:bg-accent/10`), section headers, inactive icons.

Hover state for all navbar icon links: `hover:bg-accent/10 transition-colors` — consistent with existing `uploadLinkClass` and `settingsLinkClass` patterns in `NavBar.tsx`.

## Interaction Contracts

### Navbar docs icon active state
- Condition: `location === "/docs" || location.startsWith("/docs/")`
- Active: `text-primary`
- Inactive: `text-foreground`
- Source: RESEARCH.md Pitfall 1 + existing NavBar pattern

### Sidebar active article
- Condition: `section === activeSection && slug === activeSlug`
- Active: `text-primary bg-accent font-medium`
- Inactive: `text-foreground hover:bg-accent/10`

### Route change transition
- No custom CSS transition on route changes (Claude's discretion — keep zero animation)
- Rationale: Markdown content replaces synchronously from static registry; no async fetch delay to mask

### Redirect behavior (no visible feedback)
- Bare `/docs` → role-aware redirect via `useLocation` navigate with `{ replace: true }` before render
- Admin URL visited by Viewer → silent redirect to `/docs/user-guide/intro`, `{ replace: true }`
- Guard: return `null` (not sidebar layout) while `!section || !slug` to prevent flash-of-sidebar

### Sidebar responsive collapse
- Below `md` (768px): DocsSidebar hidden (`hidden md:block`)
- No hamburger/drawer for this phase — small article count navigable via URL; mobile polish deferred
- Source: Claude's discretion (not specified in CONTEXT.md)
