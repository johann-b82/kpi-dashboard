**1. Create `backend/app/services/personio_client.py`** with the full implementation from research Pattern 4.

The file must contain:

**Constants:**
```python
PERSONIO_BASE_URL = "https://api.personio.de/v1"
TOKEN_TTL_SECONDS = 86400    # 24 hours
TOKEN_REFRESH_BUFFER = 60    # Re-auth if <60s remaining (D-13)
```

**Exception hierarchy (D-09, D-10):**
- `PersonioAPIError(Exception)` — base, with `message: str` and `status_code: int | None = None` attributes
- `PersonioAuthError(PersonioAPIError)` — for 401 responses
- `PersonioRateLimitError(PersonioAPIError)` — for 429 responses, with `retry_after: int = 60` attribute
- `PersonioNetworkError(PersonioAPIError)` — for timeout and connection errors

**PersonioClient class:**
- `__init__(self, client_id: str, client_secret: str)` — stores credentials, initializes `_token: str | None = None`, `_expires_at: float = 0.0`, creates `httpx.AsyncClient(base_url=PERSONIO_BASE_URL, timeout=30.0)` (per D-12)
- `async close(self) -> None` — calls `await self._http.aclose()`
- `async authenticate(self) -> str` — POSTs to `/auth` with `{"client_id": ..., "client_secret": ...}` JSON body. Maps responses:
  - `httpx.TimeoutException` -> `PersonioNetworkError("Personio unreachable (timeout)")`
  - `httpx.RequestError` -> `PersonioNetworkError(f"Personio unreachable: {exc}")`
  - HTTP 401 -> `PersonioAuthError("Invalid credentials")`
  - HTTP 429 -> `PersonioRateLimitError(f"Rate limited, retry in {retry_after}s", retry_after=int(resp.headers.get("Retry-After", "60")))`
  - Other non-success -> `PersonioAPIError(f"Personio auth failed with status {resp.status_code}", status_code=resp.status_code)`
  - Success: extract `resp.json()["data"]["token"]`, store in `self._token`, set `self._expires_at = time.monotonic() + TOKEN_TTL_SECONDS`, return token
- `async _get_valid_token(self) -> str` — if `self._token is None` or `time.monotonic() > self._expires_at - TOKEN_REFRESH_BUFFER` then call `await self.authenticate()`. Return `self._token`.
